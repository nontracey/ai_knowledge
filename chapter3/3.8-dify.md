# 3.8 Dify：开源 LLM 应用开发平台

## Dify 概述

### 平台定义与特性

Dify 是一个开源的 LLM 应用开发平台，与 Coze 最大区别是开源——代码免费可见，可私有化部署，确保数据安全，满足企业处理敏感信息、注重合规的需求。

Dify 定位为 LLM 应用开发平台，涵盖提示词工程、RAG 和 AI Agent 三板斧，能实现从简单文案生成到复杂超级 Agent 的多种应用。是集成开发环境，提供一站式解决方案，涵盖数据投喂、模型挑选、应用开发、效果调试到上线集成全过程。

### 核心定位

- **开源**：代码完全可见，可私有化部署
- **面向开发者**：适合构建生产级 AI 应用
- **一站式**：从开发到部署的全流程支持
- **深度控制**：提供比 Coze 更强的控制能力

## 平台特性

### 工作流设计

- **可视化画布**：像搭积木一样组合功能，构建强大清晰的 AI 工作流程
- **节点丰富**：LLM 节点、知识库节点、工具节点等
- **灵活编排**：支持复杂 AI 工作流程

### 模型支持

Dify 能与几乎所有主流大语言模型无缝集成：

| 类型 | 支持的模型 |
|------|---------|
| 闭源模型 | GPT、Claude 等 |
| 开源模型 | Llama、DeepSeek 等 |
| 自托管模型 | 本地部署的模型 |

### 提示词 IDE

专业提示词集成开发环境：精心打磨指令、对比不同模型效果、一键添加附加功能（如文字转语音）。

### RAG 引擎

提供从文档上传、智能清洗、自动分段到高效检索的全链路 RAG 功能，支持多种常见文档格式，简化私有知识库构建。

### 智能体功能

支持通过大模型函数调用或 ReAct 模式思考行动，内置超 50 种实用工具，如网页搜索、图片生成、科学计算等，还有丰富的市场资源。

### 运维监控

提供完整运维监控工具，可跟踪分析应用日志与性能，根据用户反馈和数据迭代提示词、数据集和模型，形成进化闭环。

### 后端即服务

所有核心功能配 API 接口，可作为强大 AI 后端服务，将能力集成到现有业务逻辑和应用程序中。

## Coze 与 Dify 对比

| 维度 | Coze | Dify |
|------|------|------|
| 定位 | AI Bot 构建平台，追求效率和流量 | 面向开发者的 LLM 应用开发平台，追求深度、控制力和可扩展性 |
| 开放性 | 闭源、SaaS 服务，有厂商锁定风险 | 开源，可随心修改、部署在内网，确保数据安全 |
| 模型支持 | 集成字节豆包及主流模型 | 模型无关，支持几乎所有主流商业和开源模型 |
| 核心抽象 | 以"智能体"为核心 | 以"应用"为核心 |
| 数据处理 | 有知识库 | 有数据集，支持更丰富的文本处理方式 |
| 工具/插件 | 有插件 | 有工具，支持自定义 API、代码工具和工作流工具 |
| 集成方式 | 多平台一键发布 | API 驱动，可与复杂系统深度融合 |
| 迭代与运维 | 提供对话日志分析 | 提供完整日志、标注和性能分析 |

## 服务架构解析

### 核心服务

| 服务 | 功能 |
|------|------|
| api 服务 | 后端 API 服务，是平台中枢神经，处理所有业务逻辑 |
| worker 服务 | 后台任务处理器，负责异步任务，如处理知识库文档 |
| web 服务 | 前端界面，将后端数据图形化呈现 |
| db 服务 | 主数据库，存储核心数据 |
| redis 服务 | 高速缓存和任务队列 |
| nginx 服务 | 反向代理服务器，生产环境建议开启 HTTPS |

### 向量数据库服务

RAG 的核心组件，Dify 支持多种向量数据库选择：

**原生向量数据库：**

| 数据库 | 特点 |
|--------|------|
| Weaviate | 快速构建和迭代 AI 应用 |
| Qdrant | 性能和内存安全，丰富的过滤和查询功能 |
| Milvus | 高度扩展性，支持多种索引类型，大规模生产级别 AI 应用 |
| MyScale | 向量+结构化数据，混合查询 |
| Chroma | 内存数据库，API 简单，快速验证原型 |
| Upstash | Serverless 架构，按需付费，高可用，低延迟 |

**增强型关系/NoSQL 数据库：**

- pgvector（PostgreSQL 扩展）
- Elasticsearch（通过插件支持向量搜索）
- Couchbase（NoSQL 数据库）
- Oracle（23 版本引入向量搜索）
- TiDB（分布式 SQL 数据库）

### 支撑服务

- **sandbox 服务**：代码执行沙箱，确保代码安全执行
- **plugin_daemon 服务**：插件守护进程，管理第三方插件
- **ssrf_proxy 服务**：SSRF 防护代理，防止恶意网络攻击
- **certbot 服务**：SSL 证书自动续签机器人
- **unstructured 服务**：数据处理服务，用于从非结构化文件中提取、清洗和转换数据

## 私有化部署

### 环境要求

- CPU ≥ 2 核
- 内存 ≥ 4GB
- 安装 Docker 和 Docker Compose

### 部署步骤

```bash
# 1. 获取源码
git clone https://github.com/langgenius/dify.git

# 2. 进入 docker 目录
cd dify/docker

# 3. 复制环境变量配置文件
cp .env.example .env

# 4. 启动服务
docker-compose up -d
```

### 初始化设置

访问安装页面，创建管理员账号，登录平台，进入主界面完成部署。高级配置可修改 .env 文件进行自定义配置。

## 应用类型详解

### 聊天助手应用

聊天助手是 Dify 中基础且常见的应用类型，具有持续上下文、AI 开场白、知识库集成、流式返回和 API 友好等特点。适用于智能客服、AI 伙伴、面试官/老师和企业内部知识库问答等场景。

### 文本生成应用

专注于一次性文本任务，通过表单输入变量，利用预设提示词模板快速生成内容。适用于内容创作、文本处理工具、批量任务等场景。

### Agent 应用

赋予 AI 任务分解、工具调用、自主推理能力。适用于复杂任务助理、自动化研究员、企业自动化流程等场景。

### 工作流应用

工作流是 Dify 实现"编排即应用"的工具，提供可视化编排画布。适用于后端自动化任务、批量内容生成、跨系统信息同步等场景。

### 对话流应用

围绕对话展开设计，用户输入触发流程，实现思考与回应逻辑。与工作流的区别在于：工作流面向任务自动化，对话流面向交互对话。

**对话流与工作流的区别：**

| 维度 | 工作流 | 对话流 |
|------|--------|--------|
| 核心目标 | 任务自动化，执行明确流程 | 多轮交流，构建交互式 AI |
| 交互模型 | 单向 API 驱动 | 双向多轮驱动 |
| 执行流程 | 线性批处理 | 循环事件驱动 |
| 上下文管理 | 手动管理 | 原生记忆对话历史 |
| 典型起点 | API 调用或定时触发 | 用户问候或问题 |

## Dify 对接本地/云平台大模型

### 系统推理模型

大语言模型，是 Dify 应用的核心，负责生成文本内容。例如 DeepSeek、GPT 等大语言模型，用于对话应用、内容生成等场景。

### Embedding 模型

将文本转换为数字向量，捕捉语义信息，实现语义搜索、推荐系统、文本聚类/分类，以及问答系统中的检索增强生成（RAG）。

### Rerank 模型

对搜索结果进行二次排序，提高结果相关性。在初步检索结果基础上，通过语义分析优化排名，确保提供给推理模型的上下文信息更准确。

### 语音转文本模型（ASR）

将语音转换为文本，赋予 AI 应用"听"的能力，用于语音输入、会议记录和语音指令识别。

### 文本转语音模型（TTS）

将文本转换为语音，让 AI 应用具备"说"的能力，用于语音播报、语音合成等场景。

## 模型参数介绍

### 温度（Temperature）

控制模型输出的随机性（创造力）。值越高（接近 1），模型的回答越多样化、富有创意，但也可能更不稳定或不准确。值越低（接近 0），模型的回答越确定、保守和专注。

### Top P

一种动态的采样方法，也称为"核心采样"。模型仅从累积概率超过设定值 P 的最小范围的词汇中选择下一个词。建议不和温度同时使用。

### Top K

另一种采样方法，限制候选词的数量。模型在生成每个词时，只从概率最高的 K 个词中选择。

### 存在惩罚

降低已出现过的词汇在后续生成中的概率，鼓励模型使用新词汇，减少内容重复。

### 频率惩罚

根据词条在整个生成过程中的出现频率进行惩罚，比"存在惩罚"更强，能更有效地防止重复和循环。

### 最大生成长度

以 Token 为单位，限制模型单次生成回复的最大长度。

### 跳过内容审核

绕过大模型服务商内置的内容安全过滤器。请注意：这可能导致生成有害或不安全的内容。

### 深度思考模式

开启后，会提示或要求模型在给出最终答案前，先展示其推理步骤或思考过程。

## 小结

Dify 是一个功能强大的开源 LLM 应用开发平台，适合需要深度定制和私有化部署的开发者或企业。它提供从提示词工程、RAG 到 Agent 开发的完整能力，同时保持足够的灵活性。开源特性使其特别适合对数据安全有要求的企业，而丰富的应用类型（聊天助手、文本生成、Agent、工作流、对话流）可以满足各种场景的需求。
