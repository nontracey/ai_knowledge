# 3.8 Dify 开源 LLM 应用开发平台

## 平台定义与特性

### 开源优势

Dify 开源是其与 Coze 最大区别，代码免费可见，可私有化部署，确保数据安全，满足企业处理敏感信息、注重合规的需求。

### LLM 应用定位

Dify 定位为 LLM 应用开发平台，涵盖提示词工程、RAG 和 AI Agent 三板斧，能实现从简单文案生成到复杂超级 Agent 的多种应用。

### 平台功能集成

Dify 是集成开发环境，提供一站式解决方案，涵盖数据投喂、模型挑选、应用开发、效果调试到上线集成全过程。

## 核心功能模块

### 工作流设计

提供可视化画布，像搭积木一样组合功能，构建强大清晰的 AI 工作流程。

### 模型支持

Dify 能与几乎所有主流大语言模型无缝集成，包括闭源的 GPT、Claude 和开源的 Llama、DeepSeek 等，还支持自托管模型。

### 提示词 IDE

专业提示词集成开发环境，可精心打磨指令，对比不同模型效果。

### RAG 引擎

提供从文档上传、智能清洗、自动分段到高效检索的全链路 RAG 功能。

### 智能体功能

支持通过大模型函数调用或 ReAct 模式思考行动，内置超 50 种实用工具。

### 运维监控

提供完整运维监控工具，可跟踪分析应用日志与性能。

### 后端即服务

所有核心功能配 API 接口，可作为强大 AI 后端服务。

## 服务架构解析

### 核心服务

| 服务 | 说明 |
|------|------|
| api 服务 | 后端 API 服务，是平台中枢神经 |
| worker 服务 | 后台任务处理器，负责异步任务 |
| web 服务 | 前端界面 |
| db 服务 | 主数据库 |
| redis 服务 | 高速缓存和任务队列 |
| nginx 服务 | 反向代理服务器 |

### 向量数据库选择

Dify 支持多种向量数据库：

| 类型 | 数据库 |
|------|--------|
| 原生向量数据库 | Weaviate、Qdrant、Milvus、MyScale、Chroma、Upstash |
| 增强型关系/NoSQL | pgvector、Elasticsearch、Couchbase、Oracle、TiDB |
| 云向量数据库 | 腾讯、阿里、百度、华为 |

## 应用开发流程

### 创建应用

选择应用类型，如对话型应用、文本生成型应用或 Agent 应用等。

### 调试与预览

在编排右侧的实时预览窗口与应用对话，查看应用的每一步操作和思考过程。

### 编排应用

安排应用的模型、提示词、知识库、工具和视觉等元素。

### 发布与集成

发布应用后，Dify 生成 API 接口，可将 AI 能力集成到业务系统中。

## Dify 应用类型

### 聊天助手应用

具有持续上下文、AI 开场白、知识库集成、流式返回和 API 友好等特点。

### 文本生成应用

专注于一次性文本任务，通过表单输入变量，利用预设提示词模板快速生成内容。

### Agent 智能助手

赋予 AI 任务分解、工具调用、自主推理能力，通过"思考-行动-观察"循环处理任务。

### 工作流应用

实现"编排即应用"，提供可视化编排画布，将功能节点拖拽连接，形成数据处理与逻辑执行链路。

### 对话流应用

用于构建对话式 AI 的可视化编排工具，围绕对话展开设计，用户输入触发流程，实现思考与回应逻辑。

## 对话流与工作流的区别

| 维度 | 工作流 | 对话流 |
|------|--------|--------|
| 核心目标 | 任务自动化，执行明确流程 | 交互对话，构建多轮交流 |
| 交互模型 | 单向 API 驱动 | 双向多轮驱动 |
| 执行流程 | 线性批处理 | 循环事件驱动 |
| 上下文管理 | 手动管理 | 原生记忆对话历史 |
| 适用场景 | 后端自动化任务 | 智能客服、AI 助教 |

## 模型参数介绍

- **温度（Temperature）**：控制模型输出的随机性，值越高越有创意，值越低越确定
- **Top P**：动态采样方法，从累积概率超过设定值的词汇中选择
- **Top K**：限制候选词的数量，只从概率最高的 K 个词中选择
- **存在惩罚**：降低已出现过的词汇在后续生成中的概率
- **频率惩罚**：根据词条在整个生成过程中的出现频率进行惩罚
- **最大生成长度**：限制模型单次生成回复的最大长度
