# 3.3 Agent 的发展历程与基于大模型的 Agent

## Agent 发展历程

### 符号 Agent（1950-70 年代）

最早的 AI Agent 基于符号逻辑：

- **工作原理**：使用规则和逻辑推理
- **典型代表**：专家系统，如医疗诊断系统
- **局限性**：无法应对复杂多变的现实世界，缺乏灵活性

### 反应型 Agent（1980-90 年代）

为了克服符号 AI 的局限，反应型 Agent 兴起：

- **工作原理**：根据直接刺激迅速反应
- **核心机制**：条件-动作规则（IF-THEN）
- **适用场景**：简单实时任务
- **局限性**：缺乏整体认识和长远规划能力

### 具有迁移学习和元学习能力的 Agent（2000 年代后）

21 世纪的 Agent 开始具备学习能力：

- **迁移学习**：将知识迁移到新任务
- **元学习**：自动调整学习策略
- **特点**：更强的适应性和灵活性

## 基于大语言模型的 Agent

大语言模型的突破为 Agent 带来了质的飞跃。

### 为什么大模型适合 Agent

| 优势 | 说明 |
|------|------|
| 预训练知识 | 学习到了海量的世界知识 |
| 理解能力 | 强大的语言理解和表达能力 |
| 泛化能力 | 一个模型应对多种任务 |
| 推理能力 | 支持自主决策和规划 |

### 核心逻辑框架

#### 思维链（Chain-of-Thought，CoT）

思维链引导模型逐步推理，增强逻辑能力：

```
问题：小明有5个苹果，小红给了他3个，小明吃掉了2个，小明现在有多少个苹果？

思考步骤：
1. 小明原有 5 个苹果
2. 小红给了 3 个，所以 5 + 3 = 8 个
3. 小明吃掉了 2 个，所以 8 - 2 = 6 个

答案：小明现在有 6 个苹果
```

思维链让 Agent 像人类一样，把推理过程一步一步写出来。

#### ReAct：推理与行动结合

ReAct = Reasoning + Acting，核心思想是"思考-行动-观察"循环：

```
Thought（思考）：我需要查询北京的天气来回答用户问题
Action（行动）：调用天气查询 API，参数为"北京"
Observation（观察）：API 返回"晴，25度"
...循环直到任务完成
```

#### 问题分解

将复杂任务拆解为简单子问题：

```
复杂任务：策划一次旅行
     ↓
子任务 1：查询目的地天气
子任务 2：搜索景点门票
子任务 3：预订酒店
子任务 4：规划交通路线
     ↓
整合为完整旅行计划
```

## 大模型 Agent 的优势

### 1. 预训练过程获得的能力

大模型在预训练中学习到：

- **海量世界知识**：理解各种领域常识
- **语言规律**：语法、语义、语用
- **推理模式**：逻辑推演、数学计算

### 2. 理解与表达能力

- 理解复杂多样的自然语言指令
- 生成流畅自然的回复
- 进行多轮连贯对话

### 3. 推理与自主能力

- 进行复杂的多步推理
- 根据反馈调整策略
- 自主规划行动路径

## 技术架构

典型的大模型 Agent 系统架构：

```
用户输入
    ↓
规划模块：分解任务、制定计划
    ↓
记忆模块：检索相关知识、历史
    ↓
工具模块：调用外部 API/服务
    ↓
执行模块：执行具体操作
    ↓
评估模块：检查结果、决定是否继续
    ↓
输出回复
```

## 发展原因

大模型 Agent 受到关注的原因：

1. **AI 潜力验证**：大模型证明了 AI 的巨大潜力
2. **市场需求**：各行业对智能化解决方案的需求
3. **技术成熟**：全球研究力量的共同推动

## 小结

Agent 从符号 AI 到大模型 Agent 经历了漫长发展。大语言模型为 Agent 带来了强大的理解、推理和生成能力。思维链、ReAct 等框架让 Agent 能够进行复杂推理和自主决策，未来 Agent 将成为 AI 应用的重要形态。
