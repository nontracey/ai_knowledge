# 5.1 RAG 是什么

## RAG 定义

**RAG（Retrieval-Augmented Generation，检索增强生成）** 是一种将信息检索与语言生成相结合的技术。它通过从外部知识源检索相关信息，来增强大语言模型的生成能力。

## 为什么需要 RAG

### 大模型的知识局限

大语言模型存在固有的局限性：

| 问题 | 说明 | 影响 |
|------|------|------|
| 知识过时 | 训练数据有截止日期 | 无法回答最新问题 |
| 幻觉 | 可能生成错误内容 | 影响准确性 |
| 私有知识 | 无法访问内部数据 | 应用受限 |
| 长尾知识 | 罕见信息可能被遗忘 | 覆盖不全面 |

### RAG 的解决方案

RAG 通过在生成时引入外部知识来弥补这些不足：

- **实时信息**：检索最新数据，确保时效性
- **私有知识**：访问企业内部文档
- **准确性**：基于检索到的内容生成，降低幻觉
- **可追溯**：生成内容可追溯到具体来源

## RAG 工作原理

### 两阶段流程

RAG 分为两个主要阶段：

```
┌─────────────────────────────────────────────────┐
│                  离线阶段（索引）                    │
├─────────────────────────────────────────────────┤
│  文档 → 加载 → 分割 → 嵌入 → 向量存储 → 索引           │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│                  在线阶段（检索+生成）                │
├─────────────────────────────────────────────────┤
│  用户问题 → 嵌入 → 检索 → 构建提示 → LLM生成 → 回复    │
└─────────────────────────────────────────────────┘
```

### 索引阶段

1. **加载**：从各种来源读取文档
2. **分割**：将长文档切分成小块
3. **嵌入**：将文本块转换为向量
4. **存储**：存入向量数据库

### 检索与生成阶段

1. **问题嵌入**：将用户问题转换为向量
2. **相似度检索**：在向量库中找出相关内容
3. **提示构建**：将检索结果与问题组合
4. **LLM 生成**：基于上下文生成回答

## RAG 应用场景

### 企业知识管理

- 内部文档问答
- 政策制度查询
- 产品手册检索

### 专业领域应用

- 法律咨询
- 医疗问答
- 财务报告分析

### 智能客服

- 产品问题解答
- 售后服务支持
- 个性化推荐

## RAG 的优势

| 优势 | 说明 |
|------|------|
| 知识时效性 | 可接入实时数据 |
| 准确性提升 | 基于检索内容生成 |
| 可追溯性 | 答案可追溯到来源 |
| 成本效益 | 无需微调模型 |
| 灵活性 | 随时更新知识库 |

## RAG 与微调的对比

| 维度 | RAG | 微调 |
|------|------|------|
| 知识更新 | 实时更新知识库 | 需要重新训练 |
| 成本 | 较低 | 较高 |
| 隐私保护 | 数据可本地存储 | 数据需用于训练 |
| 准确性 | 基于检索 | 基于学习 |
| 适用场景 | 知识问答 | 任务特定行为 |

## RAG 的局限性

- 检索质量依赖知识库质量
- 多跳推理能力有限
- 复杂问题处理有挑战
- 检索延迟影响响应速度

## 小结

RAG 通过将大语言模型与外部知识检索相结合，有效地解决了模型知识过时、幻觉和私有知识访问等问题。理解 RAG 的工作原理是构建高效 AI 应用的基础。
