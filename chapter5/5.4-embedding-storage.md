# 5.4 文本嵌入与向量存储

## 文本嵌入

### 什么是向量

向量是具有大小和方向的数学对象，在计算机中用有序数字列表表示：

```python
# 二维向量示例
猫 = [0.8, 0.3]  # [动物程度, 体积大小]
狗 = [0.7, 0.6]
```

### 文本嵌入原理

文本嵌入将文本映射到高维向量空间：

- **语义相似性**：语义相近的文本，向量也相近
- **向量运算**：可以通过向量距离衡量相似程度
- **维度**：通常 256-2048 维

### 嵌入模型

LangChain 提供统一的嵌入接口：

```python
from langchain_community.embeddings import HuggingFaceEmbeddings

embeddings = HuggingFaceEmbeddings(
    model_name="sentence-transformers/all-MiniLM-L6-v2"
)

# 嵌入单个文档
doc_vector = embeddings.embed_query("这是一个测试文档")

# 嵌入多个文档
doc_vectors = embeddings.embed_documents(["文档1", "文档2"])
```

### 本地嵌入模型

使用开源模型本地运行：

```python
from langchain_community.embeddings import HuggingFaceEmbeddings

model_kwargs = {"device": "cuda"}  # 使用 GPU
encode_kwargs = {"normalize_embeddings": True}

embeddings = HuggingFaceEmbeddings(
    model_name="./local-model",
    model_kwargs=model_kwargs,
    encode_kwargs=encode_kwargs
)
```

## 向量存储

### 向量数据库的作用

向量数据库是 RAG 系统的"大脑"，负责：

- 存储海量向量
- 高效相似度检索
- 元数据过滤

### 常用向量数据库对比

| 数据库 | 类型 | 优势 | 适用场景 |
|--------|------|------|---------|
| FAISS | 库 | 性能极高 | 大规模搜索 |
| ChromaDB | 数据库 | 易用性强 | 原型开发 |
| Milvus | 数据库 | 可扩展 | 企业级 |
| Qdrant | 数据库 | 过滤强大 | 混合检索 |
| pgvector | 数据库扩展 | SQL 集成 | PostgreSQL 用户 |

### FAISS 使用

Facebook AI 的高性能相似性搜索库：

```python
from langchain_community.vectorstores import FAISS

# 创建向量存储
vectorstore = FAISS.from_documents(
    documents=chunks,
    embedding=embeddings
)

# 保存到本地
vectorstore.save_local("faiss_index")

# 从本地加载
loaded_vectorstore = FAISS.load_local(
    "faiss_index",
    embeddings,
    allow_dangerous_deserialization=True
)

# 执行检索
results = vectorstore.similarity_search("查询内容", k=4)
```

### ChromaDB 使用

专为 AI 应用设计的向量数据库：

```python
from langchain_community.vectorstores import Chroma

# 创建向量存储
vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    persist_directory="./chroma_db"
)

# 持久化
vectorstore.persist()

# 检索
results = vectorstore.similarity_search("查询内容", k=4)
```

## 相似度度量

### 余弦相似度

衡量向量方向的相似性：

```
cosine_similarity = (A · B) / (|A| × |B|)
```

- 取值范围：-1 到 1
- 1 表示完全相似
- 0 表示无关联
- -1 表示完全相反

### 欧氏距离

衡量向量空间的直线距离：

```
euclidean_distance = √(Σ(Ai - Bi)²)
```

- 值越小越相似
- 受向量长度影响

### 选择建议

| 场景 | 推荐度量 |
|------|---------|
| 文本语义相似度 | 余弦相似度 |
| 图像特征匹配 | 余弦相似度 |
| 坐标/位置数据 | 欧氏距离 |

## 近似最近邻（ANN）

### 为什么需要 ANN

精确最近邻搜索需要比较所有向量，效率低下。

### ANN 算法原理

- 构建索引结构组织向量
- 检索时跳过大部分向量
- 以微小精度损失换取速度提升

### 常见 ANN 算法

| 算法 | 特点 |
|------|------|
| HNSW | 内存占用高，速度快 |
| IVF | 倒排索引，支持过滤 |
| PQ | 产品量化压缩 |

## 小结

文本嵌入将文本转换为向量表示，是 RAG 的基础。向量数据库负责存储和检索向量，需要根据规模、功能和性能需求选择合适的方案。FAISS 适合高性能需求，ChromaDB 适合快速原型开发。
