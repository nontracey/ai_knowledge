# 5.3 文档加载与文本分割

## 文档加载器

### 加载器的重要性

文档加载是 RAG 系统的第一步，负责将各种来源的数据转换为 Document 对象。

### 常用加载器对比

| 加载器 | 数据源 | 特点 |
|--------|--------|------|
| TextLoader | 纯文本 | 简单直接 |
| WebBaseLoader | 网页 | 自动解析 HTML |
| CSVLoader | CSV 文件 | 每行一个文档 |
| PyPDFLoader | PDF | 支持分页 |
| DirectoryLoader | 目录 | 批量处理 |

### TextLoader 使用

```python
from langchain_community.document_loaders import TextLoader

loader = TextLoader(
    "document.txt",
    encoding="utf-8"
)
docs = loader.load()
```

### WebBaseLoader 使用

```python
from langchain_community.document_loaders import WebBaseLoader

loader = WebBaseLoader("https://example.com/article")
docs = loader.load()
```

### CSVLoader 使用

```python
from langchain_community.document_loaders import CSVLoader

loader = CSVLoader("data.csv", source_column="url")
docs = loader.load()
```

### PDFLoader 使用

```python
from langchain_community.document_loaders import PyPDFLoader

loader = PyPDFLoader("document.pdf")
docs = loader.load()  # 每个页面一个文档
```

### DirectoryLoader 使用

```python
from langchain_community.document_loaders import DirectoryLoader

loader = DirectoryLoader(
    "./documents",
    glob="*.txt",  # 文件匹配模式
    loader_cls=TextLoader
)
docs = loader.load()
```

## 文本分割

### 为什么需要分割

- **上下文窗口限制**：模型有 Token 上限
- **检索质量**：小块便于精准匹配
- **成本控制**：减少无关内容
- **减少幻觉**：聚焦相关上下文

### 核心参数

| 参数 | 说明 | 选择建议 |
|------|------|---------|
| chunk_size | 块的目标大小 | 常见 500-2000 |
| chunk_overlap | 块之间的重叠 | 通常 0-200 |

### CharacterTextSplitter

最简单的分割器，按字符分割：

```python
from langchain.text_splitters import CharacterTextSplitter

splitter = CharacterTextSplitter(
    separator="\n\n",
    chunk_size=1000,
    chunk_overlap=200
)
```

### RecursiveCharacterTextSplitter

递归字符分割，更智能：

```python
from langchain.text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    separators=["\n\n", "\n", "。", "！", "？", ""],
    chunk_size=1000,
    chunk_overlap=200
)
```

分割逻辑：

1. 按段落分割（`\n\n`）
2. 按行分割（`\n`）
3. 按句子分割（`。！？`）
4. 按字符分割（最后手段）

### Token 分割器

按 Token 数量分割，更精确控制：

```python
from langchain.text_splitters import TokenTextSplitter

splitter = TokenTextSplitter(
    chunk_size=500,
    chunk_overlap=50
)
```

### Markdown 分割器

保留 Markdown 结构：

```python
from langchain.text_splitters import MarkdownTextSplitter

splitter = MarkdownTextSplitter(
    chunk_size=1000,
    chunk_overlap=200
)
```

### 代码分割器

针对代码文件：

```python
from langchain.text_splitters import Language

splitter = RecursiveCharacterTextSplitter.from_language(
    language=Language.PYTHON,
    chunk_size=500,
    chunk_overlap=50
)
```

## 分割策略选择

### 根据内容类型

| 内容类型 | 推荐分割器 | 块大小 |
|---------|-----------|-------|
| 纯文本 | RecursiveCharacter | 500-1000 |
| Markdown | MarkdownTextSplitter | 500-1000 |
| 代码 | Language-specific | 200-500 |
| PDF | 页面分割 | 页面级 |

### 根据检索需求

| 检索粒度 | 块大小 | 说明 |
|---------|-------|------|
| 粗粒度 | 2000+ | 保留更多上下文 |
| 细粒度 | 300-500 | 更精准匹配 |
| 混合 | 组合使用 | 平衡两者 |

### 根据模型限制

考虑模型的上下文窗口大小：

```
可用 Token = 模型上下文窗口 - 生成 Token - 提示词 Token
```

## 小结

文档加载器和分割器是 RAG 系统的重要组成部分。选择合适的加载器确保数据正确导入，选择合适的分割策略影响检索质量。实际应用中需要根据内容类型、检索需求和模型限制综合考虑。
