# 5.8 实战项目集成 MCP 设计思路

## 项目需求分析

### 确定需要集成的文件操作功能

在实战项目中，我们需要集成以下文件操作功能：

1. **配置文件读取**：读取应用配置文件
2. **日志写入**：将应用日志写入文件
3. **知识库文件访问**：RAG 应用需要读取知识库文件
4. **导出功能**：导出对话记录或分析报告

### 评估性能和安全要求

- **性能要求**：低延迟、高吞吐量
- **安全要求**：严格的文件访问权限控制，防止路径遍历攻击

### 模块划分原则

1. **MCP 客户端封装**：统一管理 MCP 客户端
2. **服务注册与发现**：管理可用的 MCP 服务
3. **请求路由**：根据请求类型路由到不同服务
4. **错误处理**：统一的错误处理和日志记录

### 接口设计规范

```python
class IMCPPervice(ABC):
    """MCP 服务接口"""
    
    @abstractmethod
    async def initialize(self) -> None:
        """初始化服务"""
        pass
    
    @abstractmethod
    async def call_tool(self, tool_name: str, params: dict) -> Any:
        """调用工具"""
        pass
    
    @abstractmethod
    async def close(self) -> None:
        """关闭服务"""
        pass

class FileSystemMCPService(IMCPServce):
    """文件系统 MCP 服务实现"""
    
    async def initialize(self) -> None:
        self.client = MCPClient()
        await self.client.connect("filesystem")
    
    async def call_tool(self, tool_name: str, params: dict) -> Any:
        return await self.client.call_tool(tool_name, params)
    
    async def close(self) -> None:
        await self.client.disconnect()
```

class MCPServiceWithFallback:
    def __init__(self, mcp_service: IMCPServce):
        self.mcp_service = mcp_service
        self.fallback_enabled = True
    
    async def call_with_fallback(self, tool_name: str, params: dict, fallback_fn):
        try:

## 核心实现

### MCP 服务管理器

```python
class MCPServiceManager:
    def __init__(self):
        self.services = {}
        self._initialized = False
    
    async def initialize(self, config: dict):
        """初始化所有 MCP 服务"""
        for service_name, service_config in config.items():
            service = self._create_service(service_name, service_config)
            await service.initialize()
            self.services[service_name] = service
        self._initialized = True
    
    async def call_tool(self, service_name: str, tool_name: str, params: dict):
        """调用指定的 MCP 服务工具"""
        service = self.services.get(service_name)
        if not service:
            raise ValueError(f"Service {service_name} not found")
        return await service.call_tool(tool_name, params)
    
    async def close(self):
        """关闭所有 MCP 服务"""
        for service in self.services.values():
            await service.close()
```

### 在 AI 层集成 MCP

```python
class MCPEnabledAI:
    def __init__(self, ai_model, mcp_manager: MCPServiceManager):
        self.ai_model = ai_model
        self.mcp_manager = mcp_manager
    
    async def process_request(self, user_input: str, context: dict):
        # 调用 AI 模型
        response = await self.ai_model.chat(user_input, context)
        
        # 检查是否需要调用 MCP 工具
        if response.requires_mcp_tool():
            tool_calls = response.get_tool_calls()
            
            results = []
            for tool_call in tool_calls:
                result = await self.mcp_manager.call_tool(
                    tool_call.service,
                    tool_call.tool,
                    tool_call.params
                )
                results.append(result)
            
            # 将 MCP 结果反馈给 AI 模型
            response = await self.ai_model.continue_chat(results)
        
        return response
```

## 最佳实践

1. **连接池管理**：复用 MCP 连接，避免频繁建立和断开
2. **超时控制**：为每个 MCP 调用设置合理的超时时间
3. **重试机制**：对临时性失败进行重试
4. **安全检查**：验证文件路径，防止路径遍历攻击
5. **资源清理**：确保在服务关闭时正确释放资源
