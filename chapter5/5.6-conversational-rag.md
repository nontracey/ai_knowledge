# 5.6 对话式 RAG 与 LangGraph

## 无状态 RAG 的局限性

### 对话场景的问题

简单 RAG 在多轮对话中表现不佳：

- **上下文丢失**：无法记忆之前的问题
- **指代理解**：无法理解"它"、"这个"等指代
- **隐含意图**：后续问题可能隐含之前的信息

### 查询改写的重要性

多轮对话中，用户问题往往依赖上下文：

```
用户：人工智能是什么？
助手：人工智能是...

用户：它有哪些应用？
助手：它广泛应用于...
```

第二个问题"它有哪些应用"需要结合第一个问题才能理解。

## 对话式 RAG 的解决方案

### 核心思路

1. **查询改写**：将依赖上下文的问题改写为独立问题
2. **历史管理**：维护对话历史
3. **上下文整合**：将改写后的问题与历史结合

### LangGraph 实现

LangGraph 专门处理循环和状态管理，非常适合对话式 RAG。

## LangGraph 核心概念

### 状态（State）

在图中共享的数据结构：

```python
from typing import List, TypedDict
from langchain_core.documents import Document
from langchain_core.messages import BaseMessage

class GraphState(TypedDict):
    messages: List[BaseMessage]          # 对话消息
    question: str                        # 当前问题
    standalone_question: str             # 改写后的问题
    documents: List[Document]             # 检索文档
    generation: str                      # 生成回答
```

### 节点（Nodes）

图中的工作单元：

```python
def rewrite_node(state: GraphState) -> dict:
    """查询改写节点"""
    question = state["question"]
    messages = state["messages"]

    # 改写逻辑
    rewritten = rewrite_question(messages, question)

    return {"standalone_question": rewritten}

def retrieve_node(state: GraphState) -> dict:
    """检索节点"""
    query = state["standalone_question"]
    docs = retriever.invoke(query)
    return {"documents": docs}

def generate_node(state: GraphState) -> dict:
    """生成节点"""
    query = state["standalone_question"]
    docs = state["documents"]
    # 生成回答
    answer = generate_response(query, docs)
    return {"generation": answer}
```

### 边（Edges）

节点间的连接：

```python
# 普通边：固定流向
workflow.add_edge("retrieve", "generate")

# 条件边：根据状态选择流向
workflow.add_conditional_edges(
    "rewrite",
    decide_to_retrieve,  # 判断函数
    {
        "retrieve": "retrieve",
        "generate": "generate"
    }
)
```

## 对话式 RAG 完整实现

### 定义状态

```python
class ConversationalState(TypedDict):
    messages: Annotated[List[BaseMessage], operator.add]
    question: str
    standalone_question: str
    documents: List[Document]
    generation: str
```

### 定义节点

```python
def rewrite(state: ConversationalState) -> dict:
    """将多轮问题改写为独立问题"""
    messages = state["messages"]
    question = state["question"]

    if len(messages) <= 1:
        standalone = question
    else:
        prompt = ChatPromptTemplate.from_messages([
            ("system", "根据对话历史改写问题为独立问题。"),
            MessagesPlaceholder("chat_history"),
            ("human", "{question}")
        ])
        chain = prompt | llm | StrOutputParser()
        standalone = chain.invoke({
            "chat_history": messages[:-1],
            "question": question
        })

    return {"standalone_question": standalone}

def retrieve(state: ConversationalState) -> dict:
    """检索相关文档"""
    query = state["standalone_question"]
    docs = compression_retriever.invoke(query)
    return {"documents": docs}

def generate(state: ConversationalState) -> dict:
    """生成回答"""
    query = state["standalone_question"]
    docs = state.get("documents", [])

    if not docs:
        prompt = ChatPromptTemplate.from_messages([
            ("system", "你是一个有帮助的助手。"),
            MessagesPlaceholder("chat_history"),
        ])
    else:
        context = "\n\n".join([doc.page_content for doc in docs])
        prompt = ChatPromptTemplate.from_messages([
            ("system", "基于以下上下文回答问题：\n{context}"),
            MessagesPlaceholder("chat_history"),
        ])

    chain = prompt | llm | StrOutputParser()
    messages = state["messages"]

    if docs:
        answer = ""
        for chunk in chain.stream({"context": context, "chat_history": messages}):
            answer += chunk
    else:
        answer = ""
        for chunk in chain.stream({"chat_history": messages}):
            answer += chunk

    return {"generation": answer, "messages": messages + [AIMessage(content=answer)]}
```

### 构建工作流

```python
from langgraph.graph import StateGraph, END

workflow = StateGraph(ConversationalState)

# 添加节点
workflow.add_node("rewrite", rewrite)
workflow.add_node("retrieve", retrieve)
workflow.add_node("generate", generate)

# 设置入口
workflow.set_entry_point("rewrite")

# 添加边
workflow.add_conditional_edges(
    "rewrite",
    should_retrieve,
    {"retrieve": "retrieve", "generate": "generate"}
)
workflow.add_edge("retrieve", "generate")
workflow.add_edge("generate", END)

# 编译
app = workflow.compile()
```

### 执行对话

```python
def run_conversation():
    messages = []

    while True:
        user_input = input("You: ")
        if user_input.lower() == "exit":
            break

        messages.append(HumanMessage(content=user_input))

        result = app.invoke({
            "messages": messages,
            "question": user_input
        })

        messages = result["messages"]
        print(f"Assistant: {result['generation']}")
```

def should_retrieve(state: ConversationalState) -> str:

## 小结

对话式 RAG 通过查询改写和状态管理解决了多轮对话的问题。LangGraph 提供了强大的状态管理和工作流控制能力，使得复杂的对话 RAG 系统得以实现。
