# 5.7 Semantic Kernel RAG

## SK RAG 架构

### 与 LangChain 的对比

| 维度 | Semantic Kernel | LangChain |
|------|-----------------|-----------|
| 设计理念 | 企业级、强类型、面向对象 | 灵活、动态、函数式 |
| 适用场景 | .NET 企业应用 | Python 快速开发 |
| RAG 封装 | 底层控制 | 高级抽象 |

### 核心概念

- **数据模型**：使用 C# 类和特性定义
- **向量存储**：强类型的向量存储抽象
- **检索服务**：ITextSearch 接口

## SK 的 RAG 生命周期

```
加载 → 分割 → 嵌入 → 索引 → 检索 → 生成
```

每个阶段都有明确的接口和实现。

## 数据模型映射

### 使用特性注解

```csharp
public class DocumentChunk
{
    [VectorStoreKey]
    public string Id { get; set; }

    [VectorStoreData]
    public string Content { get; set; }

    [VectorStoreVector(1536)]
    public ReadOnlyMemory<float> Embedding { get; set; }

    [VectorStoreData]
    public string Source { get; set; }
}
```

## 检索与生成

### 文本搜索服务

```csharp

var results = await textSearch.SearchAsync(
    "查询内容",

```

## Agentic RAG

### 传统 RAG vs Agentic RAG

| 类型 | 特点 | 适用场景 |
|------|------|---------|
| 传统 RAG | 推模式，总是先检索 | 简单问答 |
| Agentic RAG | 拉模式，自主决定是否检索 | 复杂推理 |

### Agentic RAG 实现

```csharp
// Agent 根据问题决定是否检索
var agent = new ChatCompletionAgent
{
    Kernel = kernel,
    Instructions = "你是智能助手。根据用户问题决定是否需要从知识库检索信息。"
};
```

## SK 与 LangChain 选择

### 选择 SK 的场景

- 深度集成 .NET 企业应用
- 追求稳定性和可维护性
- 需要强类型安全保障
- 团队熟悉 C#

### 选择 LangChain 的场景

- 快速原型和研究
- Python 技术栈
- 需要丰富生态系统
- 频繁调整和实验

## 小结

Semantic Kernel 提供了企业级的 RAG 实现方式，强类型和面向对象的特性使其在 .NET 环境中具有优势。Agentic RAG 则通过赋予 Agent 自主决策能力，处理更复杂的问答场景。
