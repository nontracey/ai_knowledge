# 5.2 LangChain RAG 核心流程

## LangChain 对 RAG 的支持

LangChain 提供了完整的 RAG 应用开发框架，简化了从零开发 RAG 应用的复杂性。

### LangChain 解决的痛点

从零开发 RAG 应用面临诸多挑战：

- **繁杂的数据接入**：需要对接各种数据源
- **复杂的文本处理**：文档解析、分割策略
- **多样的模型与接口**：不同模型的调用方式各异
- **专业的向量存储**：向量数据库的选型和集成
- **粘合所有环节**：各组件的无缝集成

### LangChain 的模块化架构

```
┌─────────────────────────────────────┐
│           langchain-core             │  核心接口
├─────────────────────────────────────┤
│         langchain-community          │  第三方集成
├─────────────────────────────────────┤
│  文档加载器 │ 分割器 │ 嵌入 │ 向量存储 │  组件层
└─────────────────────────────────────┘
```

## LCEL 表达式语言

LangChain 的核心表达工具，通过管道操作符 `|` 连接各个组件：

```python
# 基础 RAG 链
chain = prompt | model | output_parser
```

### LCEL 的优势

- **可组合性**：轻松组合多个组件
- **可读性**：代码清晰直观
- **流式支持**：内置流式输出
- **异步支持**：高性能异步执行
- **可观测性**：便于调试和追踪

## 文档加载器

文档加载器是 RAG 流程的入口：

```python
from langchain_community.document_loaders import TextLoader

loader = TextLoader("document.txt")
docs = loader.load()
```

### Document 对象结构

```python
class Document:
    page_content: str  # 页面内容
    metadata: dict     # 元数据（来源、页码等）
```

### 常用加载器

| 加载器 | 用途 |
|--------|------|
| TextLoader | 纯文本文件 |
| WebBaseLoader | 网页内容 |
| CSVLoader | CSV 文件 |
| PyPDFLoader | PDF 文档 |
| DirectoryLoader | 目录批量加载 |

## 文本分割器

将长文档切分成小块：

```python
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200
)
chunks = splitter.split_documents(docs)
```

### 分割策略选择

- **块大小**：根据内容类型和检索粒度选择
- **块重叠**：确保信息平滑过渡
- **分割方式**：字符、Token、语义等

## 嵌入模型

将文本转换为向量：

```python
from langchain_community.embeddings import HuggingFaceEmbeddings

embeddings = HuggingFaceEmbeddings(model_name="sentence-transformers/all-MiniLM-L6-v2")
```

## 向量存储

存储和检索向量：

```python
from langchain_community.vectorstores import FAISS

vectorstore = FAISS.from_documents(chunks, embeddings)
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})
```

## 完整 RAG 链

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough

# 定义提示模板
prompt = ChatPromptTemplate.from_messages([
    ("system", "基于以下上下文回答问题：\n{context}"),
    ("human", "{question}")
])

# 构建链
rag_chain = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt
    | model
    | output_parser
)

# 执行
result = rag_chain.invoke("什么是人工智能？")
```


## 小结

LangChain 通过模块化设计和 LCEL 表达式语言，极大地简化了 RAG 应用的开发。从文档加载、文本分割、嵌入模型到向量存储，LangChain 提供了完整的组件支持，让开发者能够快速构建高效的 RAG 应用。
