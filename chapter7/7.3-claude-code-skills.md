# 7.3 Claude Code 中使用 Skills

## Skills 在 Claude Code 中的作用

Skills 扩展了 Claude Code 的能力，使其能够调用外部工具、访问特定数据源、执行特定任务。

## 查看可用 Skills

在 Claude Code 中查看所有可用的 Skills：

```
/skills list
```

这将显示所有已安装的 Skills 及其描述。

## 常用内置 Skills

### 文件操作 Skills

- **Read**：读取文件内容
- **Write**：写入或创建文件
- **Edit**：编辑现有文件
- **Glob**：搜索文件
- **Grep**：搜索文件内容

### Git 操作 Skills

- **GitStatus**：查看 Git 状态
- **GitLog**：查看 Git 提交历史
- **GitDiff**：查看文件变更
- **GitBranch**：管理分支

### 命令执行 Skills

- **Bash**：执行 Shell 命令
- **Node**：执行 Node.js 脚本
- **Python**：执行 Python 脚本

## 使用 Skills 的方法

### 1. 直接调用

在对话中直接提到 Skill 名称：

```
请使用 Read skill 读取 config.json 文件的内容
```

### 2. 在提示词中使用

在提示词中明确指定使用的 Skill：

```
使用 Glob skill 找到所有 .py 文件，然后使用 Grep skill 搜索其中的 "import" 语句
```

### 3. 通过命令菜单

使用 `/skills` 命令打开 Skills 菜单进行选择。

## 自定义 Skills

### Skill 结构

一个自定义 Skill 通常包含：

```
my-skill/
├── skill.yaml          # Skill 定义
├── handlers/           # 处理函数
│   ├── __init__.py
│   └── handlers.py
├── knowledge/           # 知识库
│   └── docs.md
└── config.yaml          # 配置文件
```

### skill.yaml 示例

```yaml
name: my-custom-skill
version: 1.0.0
description: 我的自定义技能

tools:
  - name: my_tool
    description: 执行自定义操作
    parameters:
      - name: input
        type: string
        description: 输入参数
        required: true

handler: handlers.my_tool_handler
```

## Skills 最佳实践

### 1. 合理选择 Skill

根据任务需求选择最合适的 Skill，避免不必要的工具调用。

### 2. 提供完整参数

调用 Skill 时提供所有必要的参数，以获得准确的结果。

### 3. 检查 Skill 输出

在使用 Skill 返回的结果之前，验证其正确性。

### 4. 组合使用 Skills

复杂任务可以组合使用多个 Skills 协同完成。

## Skill 开发示例

```python
# handlers/my_tool_handler.py
from typing import Dict, Any

async def my_tool_handler(input_data: Dict[str, Any]) -> Dict[str, Any]:
    """自定义工具处理函数"""
    input_text = input_data.get("input", "")
    
    # 执行自定义逻辑
    result = process_input(input_text)
    
    return {
        "status": "success",
        "result": result
    }
```

## Skill 配置

在 Claude Code 配置文件中可以设置 Skill 相关的选项：

```json
{
  "skills": {
    "enabled": true,
    "autoLoad": true,
    "customPath": "./skills"
  }
}
```
