# 7.4 安装与管理 Skill 插件

## Skill 插件管理概述

Claude Code 支持通过插件化的方式管理 Skills，便于扩展和定制功能。

## 查看已安装的 Skills

```bash
claude-code /skills list
```

或使用命令：

```bash
claude-code skills --list
```

## 安装新 Skill

### 从官方市场安装

```bash
claude-code skills install <skill-name>
```

例如，安装 Git 相关技能：

```bash
claude-code skills install git-helpers
```

### 从本地安装

```bash
claude-code skills install --path /path/to/skill
```

### 从 GitHub 安装

```bash
claude-code skills install --github <owner/repo>
```

## 更新 Skill

### 更新单个 Skill

```bash
claude-code skills update <skill-name>
```

### 更新所有 Skills

```bash
claude-code skills update --all
```

## 卸载 Skill

```bash
claude-code skills uninstall <skill-name>
```

## 搜索可用 Skills

### 搜索官方市场

```bash
claude-code skills search <keyword>
```

例如，搜索与文件操作相关的 Skills：

```bash
claude-code skills search file
```

## Skill 配置文件

每个 Skill 都有独立的配置文件，通常位于：

```
~/.claude/skills/<skill-name>/config.yaml
```

### 配置示例

```yaml
name: file-operations
version: 1.0.0
enabled: true
settings:
  defaultEncoding: utf-8
  maxFileSize: 10485760  # 10MB
permissions:
  read: true
  write: true
  delete: false
```

## Skill 目录结构

Claude Code 的 Skills 目录结构：

```
~/.claude/
├── skills/
│   ├── skill-name-1/
│   │   ├── config.yaml
│   ├── handlers/
│   ├── knowledge/
│   └── README.md
│   └── skill-name-2/
└── config.json
```

## 权限管理

### Skill 权限级别

- **read**：只读权限
- **write**：读写权限
- **execute**：执行权限
- **admin**：管理员权限

### 配置权限

在 Skill 的 config.yaml 中配置：

```yaml
permissions:
  read: true
  write: ["*.txt", "*.md"]
  execute: false
```

## 调试 Skill

### 查看 Skill 日志

```bash
claude-code skills logs <skill-name>
```

### 测试 Skill

```bash
claude-code skills test <skill-name>
```

## 常用 Skills 推荐

### 开发辅助

- **git-helpers**：Git 操作辅助
- **docker-helpers**：Docker 操作辅助
- **code-review**：代码审查

### 文件操作

- **file-operations**：增强的文件操作
- **path-tools**：路径处理工具

### 系统管理

- **system-info**：系统信息查询
- **process-manager**：进程管理

## 常见问题

### Skill 安装失败

1. 检查网络连接
2. 确认 API Key 配置正确
3. 查看错误日志

### Skill 无法正常使用

1. 检查 Skill 是否启用
2. 确认配置文件正确
3. 查看权限设置

### Skill 版本冲突

1. 尝试更新到最新版本
2. 检查依赖兼容性
