# 2.7 提示词工程

## 什么是提示词

提示词是输入给 AI 系统的指令，可以是问题、描述或关键词，目的是引导 AI 按期望思考或创造，输出所需结果。

### 提示词的作用

提示词如同打开 AI 能力之门的钥匙，其质量直接影响 AI 输出结果的质量。好的提示词能清晰指引 AI 工作方向，激发其潜能。

## 提示词的常见形式

- **指令式提示词**：直接告诉 AI 具体要做什么
- **问答式提示词**：通过提问引导 AI 思考并回答问题
- **填空式提示词**：提供句子框架让 AI 填充内容
- **对话式提示词**：模拟人与人之间的自然对话
- **创意生成式提示词**：给予 AI 更大的发挥空间
- **思维链提示词**：通过逐步推理的方式引导 AI 分析问题

## 编写高质量提示词的关键要素

1. **明确目标**：目标越清晰、越具体，AI 越容易理解意图
2. **提供充分的背景信息**：需考虑受众特点、产品卖点、行业术语等
3. **明确输出要求**：清楚告诉 AI 输出内容的类型、格式、字数限制等

## 高级提示技巧

### 思维链提示

引导 AI 进行逐步推理，展示其思考过程，适用于需要逻辑分析的任务。

### 少样本学习

在提示词中提供少量输入-输出对的样例，帮助 AI 理解和模仿特定的风格或格式。

### 约束性提示

通过设置具体的限制条件，如字数、关键词、风格、格式等，更精准地控制 AI 的输出内容。

## 提示词框架

### TASTE 框架

| 要素 | 说明 |
|------|------|
| Task（任务） | 明确要完成的具体任务 |
| Audience（受众） | 明确目标读者或用户群体 |
| Structure（结构） | 规定输出内容的组织方式 |
| Tone（语气） | 指定语言风格 |
| Example（示例） | 提供参考样本 |

### ALIGN 框架

| 要素 | 说明 |
|------|------|
| Aim（目标） | 明确最终要达成的目标 |
| Level（难度级别） | 指定分析的深度和专业程度 |
| Input（输入） | 提供作为基础的数据或信息 |
| Guidelines（指导原则） | 规定分析应遵循的框架和标准 |
| Novelty（新颖性） | 要求加入独特见解或预测 |

### RTGO 框架

| 要素 | 说明 |
|------|------|
| Role（角色） | 设定 AI 扮演的角色 |
| Task（任务） | 描述具体要完成的任务 |
| Goal（目标） | 说明任务要达成的目标 |
| Objective（操作要求） | 规定具体的执行要求 |

### CO-STAR 框架

| 要素 | 说明 |
|------|------|
| Context（情境） | 描述背景情况 |
| Objective（目标） | 明确任务目标 |
| Style（风格） | 指定输出风格 |
| Tone（语气） | 规定语言语气 |
| Audience（受众） | 明确目标读者 |
| Response（回应） | 指定回应的具体要求 |

## 推理模型与提示词工程

### 推理模型的特点

针对推理模型（如 DeepSeek-R1）的提示词应更简洁、聚焦，多给原始信息，少规定具体步骤。

### 设计要点

- **指令直接性**：避免模糊表达，直接说明需求
- **信息聚焦性**：提供关键背景信息
- **信任模型推理能力**：描述目标而非规定步骤

## 提示词迭代技巧

1. **明确初始需求**：设定角色、结构化提问、语言简洁
2. **获取初步反馈**：用提问代替评价，多轮互动
3. **优化提示词**：增加约束条件、提供示例、使用专业术语

## 通用模型与推理模型的选择

### V3 模型特点

- 高效务实，擅长快节奏、高效率工作
- 成本亲民，性价比高
- 适用于规范化的、目标明确的任务

### R1 模型特点

- 深度思考，擅长复杂推理和策略制定
- 精准可靠，在处理复杂问题时表现更优
- 适用于开放性任务

### 组合使用

将 V3 和 R1 组合使用，实现效率与精度的完美结合。
