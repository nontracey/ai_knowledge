# 2.7 提示词工程

## 什么是提示词

### 提示词的定义

提示词（Prompt）是输入给 AI 系统的指令，可以是问题、描述或关键词，目的是引导 AI 按期望的方式思考或创造，输出所需结果。提示词如同打开 AI 能力之门的钥匙，其质量直接影响 AI 输出结果的质量。

### 提示词的作用

- 清晰指引 AI 工作方向，激发其潜能
- 引导 AI 准确把握需求，避免无用信息输出
- 高阶提示词可引导 AI 进行创新创造，突破常规思维
- 通过不断尝试和优化提示词，可了解 AI 的优势与局限

### 提示词与 AI 的关系

提示词是人类与 AI 沟通的桥梁。在与大语言模型交互时，输入的质量直接决定输出的质量——"垃圾进，垃圾出"。通过不断优化提示词，我们可以更好地理解 AI 的工作原理，与 AI 协同工作，共同创造更大价值。

## 提示词的重要性

### 对 AI 输出质量的决定性作用

输入的是垃圾，输出的也是垃圾。优质提示词能引导 AI 准确把握需求，避免无用信息输出，提升工作效率。

### 提升 AI 创新创造能力

高阶提示词可引导 AI 进行创新创造，突破常规思维。如要求 AI 设计创意产品，通过特定提示词激发其创新灵感。

### 提示词促进对 AI 的理解

通过不断尝试和优化提示词，可以了解 AI 的优势与局限。在提示词优化过程中，学会与 AI 协同工作。

## 提示词的常见形式

### 指令式提示词

直接告诉 AI 具体要做什么，适用于任务目标明确的情况：

```
写一篇关于气候变化的短文。
翻译以下句子为英文：我今天很高兴。
```

### 问答式提示词

通过提问引导 AI 思考并回答问题：

```
什么是人工智能？
解释量子计算的基本原理。
```

### 填空式提示词

提供句子框架让 AI 填充内容，有助于获得结构化且有逻辑的回答：

```
我最喜欢的动物是（），因为（）
```

### 对话式提示词

模拟人与人之间的自然对话，进行多轮交流。这种形式更适合日常沟通场景：

```
你好！能跟我聊聊今天的天气吗？
我们来讨论一下最近的电影。
```

### 创意生成式提示词

给予 AI 更大的发挥空间，鼓励其进行创作：

```
写一首描绘未来世界的诗。
设计一个创新的手机应用概念。
```

### 思维链提示词

通过逐步推理的方式引导 AI 分析问题。这种形式特别适合解决复杂问题，帮助 AI 理清思路：

```
让我们一步步来思考这个问题。首先...
```

## 编写高质量提示词的关键要素

### 1. 明确目标

写提示词的第一步是明确目标，目标越清晰、越具体，AI 越容易理解意图并给出精准回应。需明确：

- 你希望 AI 完成什么任务？
- 期望的输出是什么类型？
- 有哪些具体要求或限制？

### 2. 提供充分的背景信息

给 AI 提供足够的背景信息，使其理解更深入，输出更贴合需求。需考虑：

- 内容面向谁？受众的特点、需求、痛点是什么？
- 是否涉及特定行业或专业术语？
- 希望使用什么样的语气和风格？

### 3. 明确输出要求

清楚告诉 AI 输出内容的类型、格式、字数限制、内容侧重点、风格和语气：

- **内容类型**：文章、代码、表格、列表？
- **格式要求**：字数限制、结构框架？
- **风格语气**：正式/随意、专业/通俗？
- **侧重点**：强调什么、避免什么？

## 提升提示词质量的实用技巧

### 指令清晰化

**使用明确动词开头**，用分隔符、编号划分结构，分解复杂任务为小步骤，用关键词限定输出范围，明确指定输出格式：

```
差：看看这段代码有没有问题
好：审查以下Python代码（代码片段），重点检查：
- 代码性能优化空间
- 潜在的安全漏洞
- 代码可读性
请提供详细的改进建议，并给出优化后的代码示例。所有代码必须包含注释。
```

### 角色扮演与人设

给 AI 设定具体角色和背景，帮助其理解任务语境，输出更符合预期内容：

```
你是一位有20年教学经验的高中数学老师，擅长将复杂概念简单化。
你是一位获得过多个文学奖项的科幻小说作家，擅长世界观构建。
你是一位专注用户体验的产品经理，参与过多个独角兽公司的产品设计。
```

### 迭代优化与反馈

提示词设计是反复调整的过程。先写基础版本，根据 AI 输出结果不断优化，给出具体明确反馈，逐步接近理想效果。

## 高级提示技巧

### 思维链提示

引导 AI 进行逐步推理，展示其思考过程，特别适合用于需要逻辑分析的任务：

```
请分析这个问题，并展示你的思考过程：
1. 首先，让我们理解问题的关键信息...
2. 接下来，我们可以采用以下方法解决...
3. 最后，让我们验证答案的合理性...
```

### 少样本学习

在提示词中提供少量输入-输出对的样例，帮助 AI 理解和模仿特定的风格或格式：

```
请按照以下格式撰写产品描述：

输入：iPhone 14 Pro
输出：突破性的创新设计，卓越的性能表现。搭载A16仿生芯片，4800万像素主摄像头提供专业级拍摄体验。

输入：MacBook Air
输出：轻薄优雅的便携之选，出众的续航能力。M2芯片带来强劲性能，视网膜显示屏呈现绚丽画面。

现在，请为以下产品撰写描述：
输入：iPad Pro
```

### 约束性提示

通过设置具体的限制条件，如字数、关键词、风格、格式等，可以更精准地控制 AI 的输出内容：

```
请写一篇产品推广文案，要求：
- 字数限制在100字以内
- 必须包含关键词：创新、智能、便携
- 使用专业但不晦涩的语言
- 结构为：痛点-解决方案-产品优势
- 以号召性用语结尾
```

## 提示词工程

### 什么是提示词工程

提示词工程是一门专注于设计和优化有效提示词的学问，目的是提升 AI 输出的质量和效率。高质量的提示词能减少 AI 的试错次数，缩短迭代时间，提高内容生产效率。

### 提示词工程的核心技能

- **指令设计**：将复杂任务转化为 AI 能够理解和执行的清晰指令
- **策略选择**：根据具体任务类型和模型特点，选择合适的提示方法
- **迭代优化**：通过不断尝试、评估和反馈，逐步调整提示词

## 推理模型与提示词工程

### 推理模型的特点

针对推理模型（如 DeepSeek-R1）的提示词应更简洁、聚焦，多给原始信息，少规定具体步骤，适当设定约束条件，注重互动优化。

### 为推理模型设计提示词的要点

推理模型具备强大的推理能力，但仍需精心设计的提示词来明确任务目标、提供背景信息和设定约束条件。

**简单提示词原则：**

- **指令直接性**：
  - 不推荐：如果可以的话，我想请你帮忙看看这段代码是否存在什么问题，如果方便的话可以给出一些改进建议
  - 推荐：分析这段代码的问题并提供改进建议

- **信息聚焦性**：
  - 不推荐：我的团队有5个人，我们想做一个社交App，想听听你对产品定位的看法
  - 推荐：为面向年轻人的社交App制定产品定位策略

- **信任模型推理能力**：
  - 不推荐：首先分析用户需求，然后进行市场调研，接着制定营销策略，最后预估投资回报率。请按照这些步骤帮我评估这个商业计划
  - 推荐：评估这个商业计划的可行性

- **明确任务目标**：
  - 不推荐：帮我看看这份市场报告
  - 推荐：分析2023年Q4电商行业报告，重点关注用户增长趋势和竞品策略，给出未来6个月的市场机会

- **提供必要的背景信息**：
  ```
  设计一个支付系统。需求：日交易量10万笔，峰值QPS1000，需支持微信/支付宝/银行卡，现有技术架构和资源配置见附件。
  ```

- **设定适当的约束条件**：
  ```
  设计一个支持日订单量5000笔、响应时间不超过100ms、预算50万元的电商系统方案
  ```

## 提示词框架盘点

### TASTE 框架

TASTE 框架包含五个要素，适合初学者和多种生成任务：

- **Task（任务）**：要完成什么任务
- **Audience（受众）**：目标读者是谁
- **Structure（结构）**：文章需要包含哪些部分
- **Tone（语气）**：使用什么风格
- **Example（示例）**：参考范例

```
请帮我写一篇关于AI在教育领域应用的文章（Task）。
这篇文章面向对AI感兴趣但没有技术背景的中学教师（Audience）。
文章需要包含引言、三个主要应用场景、未来展望和总结五部分，每部分约500字（Structure）。
请使用平易近人、富有启发性的语气（Tone），可以参考《中国教育报》科技专栏的风格（Example）。
```

### ALIGN 框架

ALIGN 框架强调目标导向和难度控制，适合对输出质量要求较高的任务：

- **Aim（目标）**：要达成什么目标
- **Level（难度级别）**：分析深度需要达到什么水平
- **Input（输入）**：提供什么基础数据
- **Guidelines（指导原则）**：遵循什么分析框架
- **Novelty（新颖性）**：希望有什么独特见解

```
请帮我分析一家科技公司的财务状况（Aim）。
分析深度需要达到专业投资者水平（Level）。
我会提供该公司过去三年的财务报表数据作为基础（Input）。
分析需要遵循标准的财务分析框架，包括盈利能力、偿债能力和运营效率三个维度，并使用相关指标支持结论（Guidelines）。
希望能在分析中加入对未来发展趋势的预测，并提出独特的投资见解（Novelty）。
```

### RTGO 框架

RTGO 框架结构清晰，适合职场任务执行，角色设定提升输出专业性和可信度：

- **Role（角色）**：AI 扮演什么角色
- **Task（任务）**：要完成什么任务
- **Goal（目标）**：要达成什么目标
- **Objective（操作要求）**：有什么具体要求

```
请你扮演一位资深的社交媒体营销专家（Role），帮我为一款新上市的智能手表编写一篇微博推广文案（Task）。
这篇文案的目标是吸引25~35岁的年轻白领用户，突出产品的时尚设计和健康监测功能（Goal）。
文案要求300字以内，需包含产品卖点、用户痛点和号召性用语，语气要活泼有趣，并在文末加入2~3个相关话题标签（Objective）。
```

### CO-STAR 框架

CO-STAR 框架要素全面，适用性强，适合对输出格式有明确要求的场景：

- **Context（情境）**：背景是什么
- **Objective（目标）**：要达成什么目标
- **Style（风格）**：使用什么写作风格
- **Tone（语气）**：使用什么语气
- **Audience（受众）**：目标读者是谁
- **Response（回应）**：需要什么样的回应格式

```
我们公司是一家新成立的AI初创企业，即将举办首次产品发布会（Context）。
请帮我撰写一份产品发布会的演讲稿（Objective）。
演讲稿要体现科技感和创新精神（Style），语气要充满激情（Tone）。
演讲对象是投资人和科技媒体记者（Audience）。
演讲稿需要包含开场白、产品介绍、市场前景和结束语四个部分，每部分约500字，并配有关键数据支持（Response）。
```

## 提示词迭代技巧

### 明确初始需求

- **设定角色**：设定角色让 AI 更专业，如"资深职业规划师"，使其回答更具针对性
- **结构化提问**：通过目标、现状、障碍三方面结构化提问，让 AI 清晰了解需求
- **语言简洁**：使用简单明了的语言表达需求，避免模糊词汇

### 获取初步反馈

- **用提问代替评价**：以提问代替直接评价 AI 回答，引导其思考改进
- **多轮互动**：通过多轮互动逐步优化提示词
- **分析反馈**：仔细评估 AI 输出，找出优点和不足

### 优化提示词

- **增加约束条件**：加入格式、风格、长度等限制条件
- **提供示例**：给 AI 参考样本，明确期望输出的风格和结构
- **使用专业术语**：在熟悉领域使用专业术语，提高沟通效率

### 通用结构

提示词的通用结构包括：指令、背景、示例、约束、提问。

### 持续评估和调整

- 测试新提示词的效果
- 记录实验过程
- 总结经验，形成自己的方法

**注意：**
1. 了解 AI 的局限性
2. 识别不确定性的内容
3. 保持批判性思维

## 自动化提示词工程

### AI 自主构造提示词

自动化提示词工程让 AI 自主构造更"完美"的提示词。AI 凭借强大的语言理解和生成能力，掌握语言逻辑，自主生成精准提示词，并通过用户反馈不断迭代改进提示词生成策略。

### 如何引导 AI 构造提示词

**明确目标和任务**：清晰定义目标和任务，为 AI 提供上下文信息，使其准确理解意图。

```
请扮演一位提示词工程师，用CO-STAR框架帮我完善提示词：我想写一份关于电动汽车行业的分析报告。
```

**利用 AI 生成能力**：使用简单指令引导 AI 生成初始提示词，从不同维度和角色思考。

### 生成提示词的方法

**从角色扮演角度：**
```
请你扮演一位经验丰富的提示词导师，针对我想了解的【主题】，指导我如何一步步构建一个高效且精准的提示词。
```

**从任务类型角度：**
```
写作任务：我想用大语言模型辅助写作【文章类型】，请你为我生成一些能引导模型进行高质量写作的提示词。
问题解决：我想用大语言模型辅助解决【问题类型】，请你为我生成一些能引导模型进行深入分析的提示词。
```

**元提示**：在提示词中内置对提示词自身的要求：
```
请你先分析我的请求，指出我的请求中可能存在的歧义或不明确之处。然后，生成一个更详细、更清晰的提示词。
```

### 迭代提示词

通过量化改进法让 AI 设计评分体系并量化评估，不断优化提示词效果：

```
请为我生成一份关于【主题】的【内容类型】，生成后请你为这份内容设计一个5分制的评分体系，包含至少5个评价维度。然后，请使用你设计的评分体系，对你刚刚生成的内容进行自评打分。
```

## 其他提示词技巧

### 通俗化表达

- **说人话模式**：要求 AI 用通俗语言解释复杂概念
- **小学生模式**：让 AI 用小学生能懂的方式解释问题
- **比喻模式**：通过比喻将抽象概念与熟悉事物联系

### 思维拓展

- **穿越模式**：从未来视角看待问题，获得更长远的判断
- **杠精模式**：AI 从反面质疑观点，发现潜在问题
- **精神分裂模式**：AI 从正反两方分析问题，客观看待事物

## 推理模型与通用模型的选择

### 通用模型 V3

- **高效务实**：V3 像职场精英，擅长快节奏、高效率工作，适合日常任务
- **成本亲民**：V3 完成任务速度快，成本相对较低
- **应用广泛**：适用于规范化的、目标明确的任务

### 推理模型 R1

- **深度思考**：R1 擅长复杂推理和策略制定，适合高精度、高可靠性任务
- **精准可靠**：R1 在处理复杂问题时更精准，能提供更可靠答案
- **适用场景**：适用于开放性任务，如策略制定、复杂数据分析、代码审查

### 组合使用

- **优势互补**：将 V3 和 R1 组合使用，实现效率与精度的完美结合
- **实际应用**：如客户服务中，R1 制定退货方案，V3 快速生成邮件和查询物流

## 小结

提示词工程是与大语言模型高效交互的关键技能。通过明确目标、提供背景、规定输出格式，并运用角色扮演、少样本学习、思维链等技巧，可以显著提升 AI 输出质量。掌握 TASTE、ALIGN、RTGO、CO-STAR 等框架能帮助我们更系统地设计提示词。提示词工程是一个迭代优化的过程，需要不断尝试和调整。对于推理模型和通用模型，要根据任务性质选择合适的模型，并相应调整提示词策略。
