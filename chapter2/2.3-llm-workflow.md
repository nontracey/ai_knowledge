# 2.3 大模型工作流程

当我们向大语言模型输入一段文字，模型会经历一个复杂的过程来生成回复。本节将详细介绍这个过程的六个关键步骤。

## 第一步：信息摄入

大语言模型之所以强大，首先是因为它"读"过海量的文本。

### 海量信息的吸收

模型的训练过程就像让它阅读整个互联网的书籍、文章、网站内容。这些文本涵盖了人类知识的方方面面：科学论文、小说故事、技术文档、日常对话等。

### 信息筛选与清洗

原始的训练数据并非直接使用，而是经过精心筛选：

- **去除垃圾信息**：过滤广告、垃圾邮件、明显无意义的内容
- **敏感内容处理**：过滤违法、有害或不当内容
- **去重**：删除重复或高度相似的内容
- **质量过滤**：保留真实、丰富、多样的知识来源

### 语言直觉的形成

通过预测训练（给定前文预测下一个词），模型逐渐形成了对语言的"直觉"。这个过程类似于人类学习语言——反复练习"猜词"，逐渐掌握词语搭配、语法规则、句子结构。

但要注意：模型的"直觉"是基于统计规律的，并非真正"理解"语言。

## 第二步：拆解输入

当用户输入一段文字后，模型首先对其进行"拆解"。

### 分词：文本的数字化

模型不能直接处理文字，而是先将文本转换成模型能理解的单位——**Token**。

Token 可以是：

- 完整的词语（如"人工智能"）
- 词语的一部分（如"AI"可能被拆成"A"和"I"）
- 单个字符
- 标点符号

不同的模型使用不同的分词算法，如 WordPiece、BPE（字节对编码）等。GPT 系列使用基于字节的 BPE（Byte-level BPE），具有较强的字符泛化能力。

### Token 的数量影响

分词后的 Token 数量直接影响：

- **处理成本**：API 按 Token 计费
- **处理速度**：Token 越多，计算量越大
- **上下文占用**：Token 数量决定上下文窗口的使用

例如，中文文本通常每 1-2 个汉字对应一个 Token。

### Token 向量化

每个 Token 被转换为一个高维向量（常见维度有 768、1024、2048 等）。这个向量蕴含了丰富的语义信息：

- 词语的出现场景
- 常见的搭配词汇
- 隐含的情感色彩

向量之间的"距离"可以衡量词语语义的相似程度。

## 第三步：数字转换与语义映射

分词后的 Token 序列被转化为向量序列，并进入一个复杂的语义空间。

### 向量表示的语义理解

在这个高维空间中，每个向量都是词语语义的多维刻画。例如，"国王"和"女王"的向量在"性别"这一维度上会有明显差异，而在"地位"维度上则相近。

### 余弦相似度的作用

通过计算向量间的余弦相似度，模型能够判断：

- 哪些词语在语义上相近
- 词语之间的关联强度
- 句子想要表达的核心概念

## 第四步：注意力分析

这是 Transformer 架构的核心环节。

### 自注意力机制

模型通过自注意力机制，分析每个 Token 与序列中其他 Token 的关系强度。

以句子"我养了一只非常可爱的猫"为例：

- "猫"会高度关注"我"、"养"、"可爱"
- "非常"会关注"可爱"作为修饰关系
- "可爱的"作为一个整体关注"猫"

### 多头注意力的协作

模型使用多个注意力头同时工作，每个头关注不同的语义关系：

| 注意力头 | 关注的关系类型 |
|---------|--------------|
| 头 1 | 主语-动词关系 |
| 头 2 | 形容词-名词修饰 |
| 头 3 | 整体情感倾向 |
| 头 4 | 指代关系（它→猫） |

### 前馈神经网络

除了注意力机制，每层还包含前馈神经网络，对每个位置的表示进行非线性变换，增强表达能力。

## 第五步：深度推理

经过注意力分析后，向量序列进入多层神经网络进行"深度推理"。

### Transformer 层的堆叠

现代大模型通常有几十层甚至上百层 Transformer。每层都在前一层输出的基础上进一步提炼和整合信息：

- 第一层：捕捉基础的词语关系
- 中间层：理解短语和子句结构
- 高层：形成整体语义理解

### 残差连接与层归一化

为了训练深层网络，Transformer 使用了残差连接（将信息原封不动传递一份）和层归一化（稳定训练过程）技术。这些技术让模型能够有效地学习深层表示。

### 编码器与解码器的分工

在编码器-解码器架构中：

- **编码器**：将输入信息转换为内部表示
- **解码器**：基于编码器输出和已生成的内容，逐步生成结果

GPT 等仅解码器模型则直接进行自回归生成。

## 第六步：生成输出

最后，模型开始生成回复文本。

### 概率分布与采样

模型并不是"选择"一个词，而是为所有可能的词计算一个概率分布：

```
P(下一个词 | 输入序列 + 已生成内容)
```

这个分布代表了模型认为每个词出现的可能性。

### 生成策略

如何从这个分布中选择下一个词？常见策略包括：

1. **贪婪选择**：始终选择概率最高的词（确定但可能缺乏多样性）

2. **Top-K 采样**：从概率最高的 K 个词中随机选择

3. **Top-P 采样（核采样）**：从累积概率达到阈值 P 的最小词集中随机选择

4. **温度控制**：调整概率分布的"平滑度"，影响输出的随机性

### 终止条件

生成过程持续进行，直到满足某个终止条件：

- 生成预设的停止符号（如 `<|endoftext|>`）
- 达到最大生成长度
- 外部逻辑判断（如检测到完整回答）

## 完整流程回顾

```
用户输入 → 分词 → Token化 → 向量化
         ↓
      注意力分析 → 多头关注 → 前馈网络
         ↓
      深度推理 → 多层Transformer
         ↓
      概率计算 → 采样选择 → 生成词
         ↓
      循环直到终止
         ↓
      输出文本
```

## 统计学派的 AI 理解

理解大模型的关键在于认识到：**它本质上是一个基于统计规律的预测系统**。模型并不真正"理解"语言，而是在学习海量文本中的统计模式。

这种"统计智能"在很多任务上表现出色，但在需要真正推理、常识或事实准确性时，可能会出现错误或"幻觉"。

## 小结

大语言模型的工作流程是一个复杂但有序的过程：信息摄入建立知识储备，分词和向量化将文本数字化，注意力分析捕捉语义关联，深度推理整合信息，最终通过概率采样生成输出。理解这个过程，有助于我们更好地认识模型的能力边界，并更有效地使用它。
