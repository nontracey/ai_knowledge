# 2.3 大模型工作流程

大模型的工作流程可以分为六个阶段：信息摄入、拆解输入、数字转换、注意力分析、深度推理和生成输出。

## 1. 信息摄入

### 海量信息摄入

大模型通过阅读大量文本数据构建语言知识库，这些文本来自互联网、书籍、文章等多元化的数据源。

### 信息筛选与清洗

大模型需经过复杂细致的筛选和清洗：
- 剔除广告垃圾、敏感信息、明显错误的文本
- 保留真实、丰富、多样的知识源泉
- 避免"误食"错误、虚假内容

### 语言直觉形成

大模型通过预测训练方式，在给定一段文字的前提下猜测下一个最合理的词。通过无数次成功和失败，逐渐形成对语言的"直觉"。但这种"直觉"是基于统计式的，而非真正意义上的理解。

## 2. 拆解输入

### 输入文本拆解

大模型将输入文本拆解为更小的单位——Token。Token 可能是完整的词、词的一部分、字母或标点符号，取决于分词器设计规则。

### Token 向量转换

每个 Token 被转换为高维度的向量，如 768 维、1024 维等。这些向量蕴含丰富的语义信息，记录词的出现场景、搭配词汇、情感色彩等。通过余弦相似度可衡量词语之间的语义相似度。

### 分词技术与效率

不同大模型使用不同的分词方式：
- WordPiece（BERT 使用）
- BPE（GPT 系列使用）
- byte-level BPE（GPT-2/3/4 使用）

GPT 系列采用的 byte-level BPE 技术，能灵活应对各种字符组合。分词后的 Token 数量影响模型的处理效率和成本，token 限制成为使用大模型时需考虑的因素。

## 3. 数字转换

分词之后，token 被映射成一个个多维的向量。词语之间存在距离，这个距离可以衡量语义相似度。

## 4. 注意力分析

### 注意力机制应用

大模型通过注意力机制分析 Token 之间的关系。例如在"这只猫非常可爱"这句话中：
- "这只"和"猫"关联性强
- "猫"和"可爱"关联紧密
- "非常"对"可爱"有修饰作用

### 多头注意力机制

多头注意力机制让模型并行使用多组注意力计算，不同头关注句子的不同方面：
- 有的头专注于实体搭配
- 有的头关注情感表达
- 还有的头关注句法结构

通过多个头的协同工作，模型能够更全面、细致地理解句子的含义，形成完整的注意力矩阵，为后续推理打下基础。

## 5. 深度推理

### 深度推理过程

大模型将 Token 向量和注意力权重输入由多个 Transformer 层堆叠而成的深度神经网络进行深度推理。

### 编码器与解码器

- **编码器**：负责接收输入数据，将这些信息转换为一种可以传递的内部表示
- **解码器**：负责根据编码器传递的信息，以及之前生成的输出，逐步生成结果

### 残差连接和层归一化

- **残差连接**：信息传递过程中增加了多副本传递环节
- **层归一化**：调节器，确保每一次的输出都保持一致的尺度

## 6. 生成输出

### 输出预测与概率分布

大模型根据上文语境预测"下一个最有可能出现的词语"，基于概率分布进行预测，为每个可能的词语赋予概率值。

### 采样策略与生成控制

- **Top-k 采样**：保留概率最高的前 k 个词进行随机选择
- **Top-p 采样**：从累积概率达到设定阈值 p 的词语集合中随机选择
- **温度参数**：调节模型输出的随机性

这些策略可单独或与温度参数结合使用，实现更细腻的生成控制。

### 生成终止与优化

大模型设定特定的终止条件决定何时停止生成：
1. 生成了预设的停止符号
2. 生成的内容达到最大长度限制
3. 外部逻辑判断停止

## 本质思考

大模型的本质还是统计式的——建立在概率模式之上的总结。
