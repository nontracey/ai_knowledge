# 2.9 大模型云平台与本地部署

## 云平台计费模式解析

### Token 的定义与作用

Token 是文本中的"语义单元"，用于模型处理语言信息。不同模型分词器不同，导致 Token 数量可能有差异。

### Token 与计费关系

Token 数量反映模型处理复杂程度，是衡量工作量的重要指标：
- 输入 Token：接受和预处理数据，消耗较小
- 输出 Token：思考并生成答案，消耗较大
- 输出 Token 价格通常高于输入 Token

### 模型的规格

- 64K = 64000 个 Token
- 8K = 8000 个 Token

## 本地部署大模型

### 单机简单部署

#### Ollama 的作用

Ollama 是开源框架，专注于本地运行大模型，提供模型管理、资源调度、性能优化等功能。

#### 部署流程

1. 官网下载安装 Ollama
2. 找到合适模型，使用命令部署
3. 终端窗口试用
4. 通过 .NET 或其他语言调用

### 本地部署模型选型与量化

#### 模型选型要点

- **硬件与任务匹配**：根据硬件配置和任务需求选择模型
- **部署环境考量**：数据敏感场景选本地服务器，需弹性扩展选云部署
- **模型优化方法**：量化可将 32 位浮点数精度降低到 8 位整数，减少内存占用约 70%

#### 模型量化详解

- **INT8 和 INT4**：常见量化方式，可将模型体积缩小四分之一，推理速度提升 2~3 倍
- **块级量化**：将权重分块量化，减少量化误差
- **动态量化与 QAT**：动态量化可根据设备状态调整量化程度

### LM Studio 部署流程

LM Studio 提供友好的界面和更多实用工具：
- 基于 llama.cpp
- 支持抱脸网的开源模型
- 可观察模型的量化精度
- 提供部署建议和监控本地资源利用
- 支持 OpenAI 接口

### 本地部署与云平台的选择

| 考量因素 | 本地部署 | 云平台 |
|---------|---------|--------|
| 模型数量 | 少 | 多 |
| 数据保密性 | 强 | 取决于平台 |
| 资源要求 | 高 | 按需付费 |
| 维护成本 | 高 | 低 |
