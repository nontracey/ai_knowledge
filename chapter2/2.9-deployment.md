# 2.9 大模型云平台与本地部署

## 云平台使用

### 计费模式解析

大模型云平台通常采用 **Token** 作为计费单位：

- **Token 定义**：Token 是模型处理文本的基本语义单元，可能是词、词的一部分或标点
- **计费方式**：输入和输出分别计费，通常输出 Token 价格更高
- **价格差异**：不同模型、不同长度的 Token 价格差异很大

### 上下文窗口

模型的规格通常用"最大上下文长度"来表示：

| 规格 | Token 数量 | 约等于 |
|------|-----------|-------|
| 8K | 8,000 | 万字左右 |
| 32K | 32,000 | - |
| 64K | 64,000 | - |
| 128K | 128,000 | 约 20 万汉字 |
| 1M | 1,000,000 | 百万级 |

上下文窗口越大，模型能够处理的文本越长，但成本也越高。

## 本地部署方案

当云服务无法满足需求时，可以考虑本地部署。

### Ollama：简化本地运行

Ollama 是一个开源框架，专注于在本地运行大模型：

- 提供模型管理功能
- 简化部署流程
- 支持多种模型
- 跨平台支持（Mac/Linux/Windows）

**部署流程：**

```
1. 下载安装 Ollama
2. 搜索并下载适合的模型
3. 终端直接运行
4. 通过 API 调用
```

### 模型选型要点

#### 硬件与任务匹配

| 硬件配置 | 推荐模型 | 说明 |
|---------|---------|------|
| 普通 PC | 7B 以下 | 智能客服类应用 |
| 高配 PC/工作站 | 13B-33B | 平衡性能和资源 |
| 服务器/GPU 集群 | 70B+ | 复杂推理任务 |

#### 任务类型适配

- **对话/问答**：GPT 类模型即可
- **专业领域**：可微调垂直领域模型
- **视觉任务**：选择视觉语言模型
- **资源受限**：选择蒸馏/量化模型

### 模型量化

量化是将模型从高精度（如 32 位浮点）压缩到低精度（如 8 位整数）的技术。

#### 常见量化方式

| 量化级别 | 精度 | 体积压缩 | 性能保留 |
|---------|------|---------|---------|
| FP16 | 16 位 | 50% | 接近无损 |
| INT8 | 8 位整数 | 75% | 大部分保留 |
| INT4 | 4 位整数 | 87.5% | 明显下降 |

#### 量化技术详解

- **块级量化**：将权重分块量化，减少误差
- **动态量化**：根据设备状态调整量化程度
- **QAT（量化感知训练）**：在训练时模拟量化效果

#### 量化效果

INT4 量化后，模型体积可缩小到原来的 1/4，推理速度提升 2-3 倍，但输出质量可能有所下降。

### LM Studio：友好的本地部署工具

LM Studio 是一款用户友好的本地大模型运行工具：

- 基于 llama.cpp 引擎
- 提供图形界面
- 支持多种模型格式
- 内置模型量化功能
- 提供 OpenAI 兼容的 API

**功能特点：**

- 模型下载和管理
- 对话交互界面
- 服务器模式（提供 API）
- 资源监控
- 支持 GGUF 格式模型

## 本地部署 vs 云平台

| 对比维度 | 本地部署 | 云平台 |
|---------|---------|-------|
| 数据隐私 | 完全可控 | 需上传数据 |
| 成本 | 硬件一次投入 | 按量付费 |
| 可用模型 | 取决于硬件 | 多种选择 |
| 维护成本 | 自行维护 | 托管服务 |
| 延迟 | 本地低延迟 | 依赖网络 |
| 规模 | 受硬件限制 | 弹性扩展 |

## 选择建议

**选择本地部署的场景：**

- 数据敏感，不能上传到第三方
- 使用频率高，API 成本累计高
- 需要深度定制和优化
- 离线环境使用

**选择云平台的场景：**

- 快速验证想法
- 流量波动大
- 缺乏运维能力
- 需要最新模型能力

## 小结

大模型的部署方式各有优劣。云平台适合快速起步和弹性需求，本地部署适合数据安全和成本敏感场景。Ollama 和 LM Studio 等工具降低了本地部署的门槛，量化技术让我们在有限硬件上运行大模型成为可能。
