# 2.1 向量化与词向量

## 为什么需要向量化

计算机最擅长处理数字，而人类语言是符号化的文字。如何让计算机"理解"人类语言？答案是将语言转化为数字表示——这就是向量化的核心思想。

### 从编码到向量化

早期的计算机使用简单的编码方式来表示字符。例如 ASCII 编码中，字母"A"被编码为数字 65，"B"为 66。这种编码方式让计算机能够存储和处理文本，但存在明显局限：

- **无法表达语义**：编码只是符号的编号，"猫"和"狗"在编码层面毫无关联
- **缺乏上下文理解**：同一个词在不同语境下的含义无法区分

举个简单的例子，如果我们只用编码来表示词语：

```
猫 → 2001
狗 → 2002
动物 → 3001
```

虽然计算机知道"猫"和"狗"都是动物，但这种关联需要人工设定，无法自动发现。

## 什么是词向量

词向量（Word Vector）是一种将词语表示为高维数值向量的技术。它的核心思想是：**用一组数字（向量）来描述一个词语的含义**。

### 词向量的直观理解

假设我们要用一个三维向量来表示词语的以下特征：

- 动物程度（0-100）
- 食肉程度（0-100）
- 体型大小（0-100）

那么：

```
猫 = [90, 85, 30]  # 高动物程度，高食肉程度，中等体型
狗 = [90, 70, 50]  # 高动物程度，中等食肉程度，中大体型
动物 = [100, 50, 50]  # 完全是动物，食肉/食草中等，体型中等
苹果 = [0, 0, 10]   # 不是动物，小体型
```

通过这样的表示，我们可以计算词语之间的"距离"。语义相近的词语，其向量也相近。

## 句子向量化的概念

单个词语的向量是基础，但真正的理解需要处理完整的句子。**句子向量化**是将整个句子转化为一个向量，这个向量综合了句子中所有词语的信息，代表整句话的语义。

例如：

```
句子1：我养了一只猫
句子2：我养了一只狗
句子3：今天天气很好
```

句子1和句子2的向量会比较接近，因为它们都描述了"养宠物"这个概念。句子3的向量则会明显不同。

## 向量运算与语义关系

词向量的一个神奇特性是，可以通过向量运算来发现词语之间的语义关系。这就是著名的"国王-男人+女人≈女王"例子：

```
向量(国王) - 向量(男人) + 向量(女人) ≈ 向量(女王)
```

这是因为在向量空间中，性别这一维度的差异被正确地捕捉和迁移了。

## 余弦相似度：衡量语义距离

如何衡量两个向量（从而两个词语或句子）的相似程度？常用的方法是**余弦相似度（Cosine Similarity）**。

余弦相似度计算两个向量夹角的余弦值：

- 当两个向量方向完全相同时，夹角为 0°，余弦值为 1（完全相似）
- 当两个向量垂直时，夹角为 90°，余弦值为 0（无关联）
- 当两个向量方向相反时，夹角为 180°，余弦值为 -1（完全相反）

```
余弦相似度 = (A · B) / (|A| × |B|)
```

在自然语言处理中，余弦相似度被广泛用于：

- 语义搜索：找出与查询语义最相似的文档
- 文本聚类：将语义相近的文本归为一类
- 问答系统：找到与问题最匹配的答案

## 大语言模型与向量化

大语言模型（LLM）是建立在向量化基础之上的。当我们向 ChatGPT 提问时，背后发生的事情是：

1. **输入向量化**：问题文本被转化为向量
2. **语义理解**：模型通过神经网络处理这个向量，理解问题含义
3. **生成输出**：模型生成回答的向量表示
4. **向量解码**：最后将回答的向量转化回人类可读的文本

整个过程涉及成千上万维的向量空间，模型在这个空间中穿梭，完成复杂的语义理解和生成任务。

## 嵌入模型

负责将文本转化为向量的模型叫做**嵌入模型（Embedding Model）**。在 RAG（检索增强生成）等应用场景中，嵌入模型起着关键作用：

- **文档嵌入**：将知识库的文档转化为向量
- **查询嵌入**：将用户问题转化为向量
- **相似度搜索**：在向量空间中找出与问题最相关的文档

不同的嵌入模型会产生不同质量的向量，直接影响最终应用的效果。常用的嵌入模型包括 OpenAI 的 text-embedding 系列、BAAI 的 bge 系列，以及国内阿里的文本嵌入模型等。

## 小结

向量化是让计算机理解人类语言的关键技术。通过将词语映射到高维向量空间，语义相近的词语在空间中距离相近，使得计算机能够进行语义计算。大语言模型正是在向量化的基础上，通过深度神经网络实现了对语言的深刻理解。理解向量化原理，有助于我们更好地使用和优化 AI 应用。
