# 2.11 Cursor AI 编程助手

## Cursor 概述

Cursor 是一款 AI 驱动的代码编辑器，基于 VS Code 构建，集成了先进的大语言模型能力，为开发者提供智能编程辅助。核心价值在于将重复性、机械性的工作交给 AI 处理，让开发者专注于更有创造性的工作。

### 核心价值

Cursor 能够帮助开发者：

- **需求分析阶段**：通过自然语言转化为技术方案
- **编码阶段**：智能补全、写代码、创建项目结构
- **测试阶段**：自动生成测试用例、分析代码覆盖率
- **后期维护阶段**：代码重构、性能调优、自动生成技术文档

### 一边查询文档，一边记语法

Cursor 的独特之处在于它能够同时处理代码编写和文档查询，帮助开发者快速理解和掌握新的编程语法与库函数。

## 安装与易用性配置

### 下载安装

访问 cursor.so 下载对应平台的安装包。

### Windows 用户配置

在设置里面功能 → 终端，设置默认终端 profile 为 Git Bash，不然容易执行失败。

### 模型配置

在 Settings → Models 中设置免费模型或付费模型。免费模型适合日常使用，Pro 模型具有更强的代码理解和生成能力。

## 开发项目流程

### 项目创建

使用 Cursor 创建项目时，需新建文件夹并初始化 Git 仓库，然后通过 AI 对话面板与 Cursor 交互。

### 三种工作模式

| 模式 | 适用场景 | 特点 |
|------|---------|------|
| **Agent 模式** | 复杂任务自动化执行 | 理解高层需求，自动执行多步骤操作 |
| **Ask 模式** | 理解代码逻辑 | 解释代码功能，回答技术问题 |
| **Manual 模式** | 精确编辑 | 精准控制 AI 修改的范围和内容 |

### 使用 Agent 模式

输入详细提示词，Cursor 会生成项目结构和基础代码，可选择接受、拒绝或重新生成修改。

**提示词示例：**

```
建一个响应式卡片式新闻网贴的基础项目结构，要求如下：

项目结构应包括：
- index.html：主页面，包含基本的HTML5结构和响应式设计的meta标签，页面展示新闻卡片，每个卡片应包括图片、标题、摘要、原文跳转链接，整体布局采用卡片式设计
- style.css：样式文件，包含基本的CSS Reset样式、卡片组件的样式设计（阴影、圆角、图片比例控制、悬停动画等）、支持主题色切换（默认主题为"中国红"）
- main.js：JavaScript文件，包含基本的DOM操作示例、动态加载新闻数据并渲染卡片、页面支持用户选择主题色并实时切换
- images/：图像资源文件夹
- README.md：项目说明文档

其他要求：
- 使用语义化HTML标签（如<section>、<article>、<header>等）
- 页面布局适配PC和移动端，推荐使用Flexbox或CSS Grid实现响应式
- 新闻摘要限制为两到三行，超出部分使用省略号处理
```

## 代码补全

### 补全和自动导入设置

在 Settings → Features 中启用：

- **Inline Completion**：内联代码补全
- **Auto Import**：自动导入
- **Ghost Text**：幽灵文本（预测性补全提示）

## 进阶项目开发实战

### 销售数据分析系统

实战项目：读取 CSV（销售数据）→ 使用 Pandas 库进行数据清洗、汇总、统计 → 前后端分离 → 前端负责可视化，后端业务逻辑（FastAPI）。

### 开发步骤

1. 编写开发目标，生成架构方案
2. 生成开发文档
3. 使用开发文档指导 Cursor 开发
4. 指导 Cursor 进行阶段性测试

### 沟通技巧

- **明确描述需求**：越具体越好
- **分步骤引导**：复杂任务分解为小步骤
- **提供上下文信息**：上传相关文件或截图
- **迭代优化**：初次生成不完美是正常的，通过反馈逐步改进
- **有效反馈**：指出需要修改的具体部分

### 提示词对比

好的提示词应该具体、明确，包含技术栈、布局要求、交互细节等。差的提示词模糊、笼统，无法指导 AI 生成准确的结果。

## 规则系统

### 使用自己的开发规范

通过 `.cursorrules` 文件定义项目开发规范：

```
# 代码风格
- 使用中文注释
- 变量命名采用 camelCase
- 函数单一职责

# 技术规范
- 使用 fetch 进行 API 请求
- 错误处理必须完善
- 所有 API 调用需加 try-catch
```

## Cursor 最佳实践

### 1. 明确需求描述

提供具体的技术要求和约束条件，避免模糊表述。好的提示词应包含：技术栈、页面结构、功能需求、交互细节等。

### 2. 分步骤进行

复杂任务分解为多个简单步骤，逐步完成。避免一次性要求 AI 完成整个项目，而是分阶段给出指令。

### 3. 提供上下文

上传相关文件或截图，帮助 AI 更好地理解需求。在开始新功能前，可以先让 Cursor 阅读现有代码。

### 4. 迭代优化

初次生成不完美是正常的，通过反馈逐步改进。指出具体需要修改的地方，而不是推翻重来。

### 5. 代码审查

AI 生成的代码仍需开发者审查，确保质量和安全性。特别是在处理用户输入、数据库操作等关键环节。

## 应用场景

### 快速原型开发

适合 Hackathon 或快速验证想法的场景，Cursor 可以快速生成可运行的项目框架。

### 学习新技术

通过让 AI 解释代码，可以快速理解新框架或新技术的使用方式。

### 代码重构

上传旧代码，让 AI 分析并提出重构建议。

### 技术文档生成

为现有代码生成注释和文档。

## 小结

Cursor 是 AI 编程时代的利器，通过 Agent、Ask、Manual 三种模式，覆盖了从需求分析到代码维护的全流程。其核心价值在于将重复性、机械性的工作交给 AI 处理，让开发者专注于更有创造性的工作。合理使用 Cursor 可以显著提升开发效率，但需要注意 AI 生成代码的质量把控，开发者仍需对代码质量和安全性负责。
