# 4.13 AutoGen 多智能体框架

## AutoGen 框架概览

### 框架简介

由微软推出的开源框架，专为构建基于 LLM 的复杂应用而设计。核心思想是通过智能体间的对话与协作来动态驱动任务解决。

### 与 LangChain 和 SK 对比

- **LangChain**：通过模块化组件编排任务逻辑形成"链"或"图"
- **Semantic Kernel**：侧重将 LLM 能力集成到企业级应用
- **AutoGen**：核心抽象是"对话"，通过智能体协作隐式驱动任务解决

## AgentChat 基础概念

### 可对话智能体

可对话智能体是一个能够发送和接收消息并基于消息产生回应的实体，是状态化和消息驱动的。

### 核心特性

1. **消息驱动**：所有行为由消息触发
2. **状态化**：在多次交互中维持内部状态
3. **可对话**：以消息收发为核心交互方式

### 预设智能体角色

- **助手智能体**：执行任务的核心力量，由大语言模型驱动
- **用户代理智能体**：人类用户的代理人，负责发起对话并代表用户执行操作

## AgentChat 交互模式与工作流

### 双智能体对话模式

模拟两个人之间的一对一交流，由一个智能体发起对话，接收方根据配置生成回复，形成轮流发言的循环。

### 顺序对话模式

一连串双智能体对话的链式组合，前一个对话的输出成为后一个对话的输入。

### 群聊对话模式

允许多个智能体在一个共享对话空间中交流，引入群聊管理器作为核心角色。

## 团队与多智能体协作

### AgentChat 团队

一组智能体集合，为了达成共同目标而协同工作。通过多智能体协作，能够有效克服单一智能体的性能瓶颈。

### 轮询式小组对话

所有成员共享同一个对话上下文，并按顺序轮流发言。

### 选择式小组对话

通过大语言模型的"选择器"动态决定下一个发言的智能体。

| 特性 | 轮询式 | 选择式 |
|------|--------|--------|
| 发言者选择 | 基于预设列表顺序 | 基于 LLM 上下文理解 |
| 对话流程 | 线性、可预测 | 动态、自适应 |
| 适用场景 | 流程固定的任务 | 流程不确定的复杂任务 |

## 用户代理智能体实现人机协作

### 人机协作的必要性

在某些任务中，人类的监督、决策和领域知识至关重要。

### 场景

- 监督与纠偏
- 提供外部信息
- 处理模糊性
- 最终审批

## Swarm 集群团队模式

### 核心思想

去中心化任务传递，每个智能体通过 HandoffMessage 主动将任务传递给下一个合适的协作者。

## Magentic-One 系统

### 定位与价值

基于 AutoGen 框架构建的预构建通用任务解决专家系统，具备处理复杂任务的能力。

### 编排者双循环机制

- **内循环**：围绕进度分类账进行任务执行与即时反思
- **外循环**：当内循环陷入停滞时，重新审视任务分类账，调整宏观计划

### 专家角色能力矩阵

| 角色 | 能力 |
|------|------|
| WebSurfer | 网页浏览与交互 |
| FileSurfer | 本地文件系统操作 |
| Coder | 代码编写和数据分析 |
| ComputerTerminal | 安全的代码执行环境 |

## GraphFlow 工作流编排

### 设计哲学

确定性流程引擎，通过有向图精确控制智能体执行顺序。

### 核心组件

- **DiGraphBuilder**：构建工作流蓝图
- **Node**：代表智能体
- **Edge**：定义节点间的执行依赖关系

### 顺序工作流

按既定顺序传递任务。

### 并行分流与合流

多个智能体并行处理任务，最终合流。

### 条件边与动态路由

通过 lambda 条件函数实现动态路由控制。
