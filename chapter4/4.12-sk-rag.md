# 4.12 Semantic Kernel RAG

## SK RAG 架构与抽象层

### 抽象层转变

Semantic Kernel 为 RAG 应用提供了现代化、企业级的抽象层，从单一组件向精细、解耦、类型安全的抽象转变。

### 数据模型映射

通过定义 C# 类和添加特性，可以将数据模型映射到向量数据库的模式：

```csharp
public class DocumentChunk
{
    [VectorStoreKey]
    public string Id { get; set; }
    
    [VectorStoreData]
    public string Content { get; set; }
    
    [VectorStoreVector(1536)]
    public ReadOnlyMemory<float> Embedding { get; set; }
}
```

## RAG 完整生命周期

RAG 应用的核心生命周期包括：

1. **数据加载与预处理**
2. **文本分块**
3. **嵌入**
4. **索引**
5. **检索**
6. **增强生成**

## 文本预处理与分块

### 文本分块的重要性

将长文本分割成适合嵌入和检索的小块，提高检索的准确性和效率。

### 自定义 Token 计算器

默认的 Token 计算器在处理中文分词时存在局限性，通过自定义 Token 计算器可以解决这一问题。

### TextChunk 静态类

Semantic Kernel 提供了 TextChunk 静态类，包含按行分割和段落组合等核心方法。

## 向量存储与模型映射

### 向量存储设计

通过定义 C# 类和添加特性，声明性地描述数据模型如何映射到向量数据库的模式。

### 核心特性

- `VectorStoreKey`：主键
- `VectorStoreData`：数据字段
- `VectorStoreVector`：向量字段

## 文本搜索服务

### ITextSearch 接口

Semantic Kernel 提供了 ITextSearch 接口，将复杂的向量检索过程封装成一个简单的基于自然语言的搜索操作。

## SK 构建标准 RAG 应用流程

```csharp
// 1. 加载文档
var documents = await File.ReadAllTextAsync("document.txt");

// 2. 文本分割
var chunks = TextChunk.SplitByToken(document, maxTokens: 512);

// 3. 嵌入
var embeddings = await embeddingService.EmbedAsync(chunks);

// 4. 存储到向量数据库
await vectorStore.UpsertAsync(chunks, embeddings);

// 5. 检索
var results = await vectorStore.SearchAsync(query, topK: 5);

// 6. 生成
var response = await kernel.InvokePromptAsync($"基于以下内容回答：{results}");
```

## SK RAG 与 LangChain RAG 对比

| 特性/组件 | Semantic Kernel | LangChain |
|-----------|-----------------|-----------|
| 数据加载 | 需手动编码 | 庞大 DocumentLoader 生态 |
| 文本分块 | TextChunk 静态类 | 丰富的 TextSplitter |
| 向量存储 | 强类型，与 C# 数据模型绑定 | 统一向量存储包装器接口 |
| RAG 流程构建 | 手动流程 | 高度封装 |
| 数据接入便捷性 | 需开发者自行实现 | 开箱即用 |

## 设计理念对比

- **Semantic Kernel**：强调强类型、面向对象、依赖注入和关注点分离，适合构建长期维护的生产级 AI 应用
- **LangChain**：侧重灵活性和快速实验，采用函数式、声明式的编程范式

## Agentic RAG

### 传统 RAG 局限性

传统 RAG 缺乏动态决策能力，无法处理模糊查询和多步推理任务。

### Agentic RAG 核心思想

引入智能体的概念，通过推理与规划、工具选择与使用、迭代执行来处理更复杂的任务。

### 智能 RAG 与传统 RAG 对比

| 类型 | 模式 | 说明 |
|------|------|------|
| 标准 RAG | 推模式 | 总是先执行检索、推送给语言模型 |
| 智能 RAG | 拉模式 | Agent 根据内部推理自主决定是否需要从知识库拉取信息 |

### TextSearchProvider 实现

TextSearchProvider 实现了 IAIContextProvider 接口，在 Agent 调用语言模型之前被触发，分析用户输入并进行相关性搜索。
