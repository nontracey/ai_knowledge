# 4.11 Semantic Kernel 流程框架

## 核心思想与架构解析

### 应用场景与优势

当插件数量多时，单纯依靠大语言模型编排函数调用不可靠，流程框架可解决此挑战。

### 关键转变

从"让 AI 决定做什么"转变为"让 AI 高效执行预定义流程"，开发者通过声明式语法控制业务流程。

### 架构基础：事件驱动架构

- **高度解耦与模块化**：每个步骤独立开发、测试和替换
- **异步与并发**：步骤发布事件后无需等待响应
- **灵活性与可扩展性**：修改事件路由规则或增加新步骤即可适应业务变更

### 核心概念

| 概念 | 说明 |
|------|------|
| 流程（Process） | 顶层容器，由步骤和事件路由规则构成 |
| 步骤（Step） | 流程的原子性活动，封装具体业务逻辑 |
| 内核函数 | 步骤类中用 [KernelFunction] 标记的方法 |
| 事件（Event） | 特定类表示，包含 Id 和 Data 属性 |
| 事件路由 | 基于 Id 决定事件流向 |
| 上下文 | 提供运行时信息和提交事件的方法 |

## 构建第一个线性流程

### 定义流程步骤

```csharp
public class CollectInfoStep : KernelProcessStep
{
    [KernelFunction]
    public async Task<ProductInfo> CollectProductInfo(KernelProcessStepContext context)
    {
        // 收集产品信息
        return new ProductInfo { Name = "示例产品" };
    }
}
```

### 编排工作流

```csharp
using Microsoft.SemanticKernel.Process.Runti e;

var builder = new ProcessBuilder("ProductWorkflow");

var collectStep = builder.AddStep<CollectInfoStep>();
var generateStep = builder.AddStep<GenerateDocStep>();
var publishStep = builder.AddStep<PublishDocStep>();

collectStep.Name("collect")
    .OnEvent(ProductEvents.InfoCollected).SendTo(generateStep);
generateStep.Name("generate")
    .OnEvent(ProductEvents.DocGenerated).SendTo(publishStep);

var process = builder.Build();
await process.StartAsync(kernel, new KernelProcessContext());
```

## 在流程中集成 AI 服务

```csharp
public class AIGenerateDocumentationStep : KernelProcessStep
{
    [KernelFunction]
    public async Task<string> GenerateDocumentation(
        KernelProcessStepContext context,
        ProductInfo product)
    {
        // 调用 AI 模型生成文档
        var result = await kernel.InvokePromptAsync($"为{product.Name}生成文档");
        return result.ToString();
    }
}
```

## 高级流程编排功能

### 实现有状态的步骤

复杂场景中，步骤需维护内部状态并在多次调用间持续存在。

### 自定义事件与条件分支

```csharp
public class ReviewStep : KernelProcessStep
{
    [KernelFunction]
    public async Task HandleReview(KernelProcessStepContext context, Document doc)
    {
        bool approved = await ReviewDocument(doc);
        
        if (approved)
            context.EmitEvent(new ProcessEvent("Approved", doc));
        else
            context.EmitEvent(new ProcessEvent("Rejected", doc));
    }
}

builder.OnEvent("Approved").SendTo(publishStep);
builder.OnEvent("Rejected").SendTo(revisionStep);
```

### 人机协作模式

```csharp
// 代理步骤：流程向外发送信号
builder.AddStepFromType<HumanApprovalStep>();

// 空闲状态等待外部输入
await process.StartAsync(kernel, new KernelProcessContext());
```

## 流程框架与智能体框架对比

| 特性 | 流程框架 | 智能体框架 |
|------|----------|------------|
| 关注点 | 确定性 | 自主性 |
| 执行路径 | 完全可预测 | 动态生成 |
| 适用场景 | 企业级业务流程 | 复杂问题解决 |

## 混合模式

兼具智能体灵活性和流程可靠性：
- 智能体分解任务
- 流程执行标准化子任务
- 反馈结果

## 最佳实践

1. **保持步骤单一职责**：设计良好步骤应只关注一件事
2. **定义清晰事件契约**：使用常量或枚举定义事件 ID
3. **合理设计状态对象**：状态对象应轻量化
4. **隔离 AI 与业务逻辑**：分离确定性业务逻辑与非确定性 AI 生成逻辑
5. **优先使用函数结果传递**：简单线性流程优先使用 `OnFunctionResult()`
