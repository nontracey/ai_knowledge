# 4.8 Semantic Kernel 插件与函数

## 插件概述

插件是 Semantic Kernel 扩展 AI 能力的核心机制。

### 插件与函数的关系

```
插件（Plugin）
├── 函数1（Function）
├── 函数2（Function）
└── 函数3（Function）
```

- **插件**：函数的逻辑分组
- **函数**：插件的基本单位

### 两类函数

| 类型 | 驱动方式 | 定义方式 |
|------|---------|---------|
| 语义函数 | 语言模型 | 提示词模板 |
| 原生函数 | C# 代码 | [KernelFunction] 特性 |

## 语义函数

### 标准文件结构

```
Plugins/
└── TravelPlugin/
    ├── Suggestion/
    │   ├── skprompt.txt
    │   └── config.json
    └── Book/
        ├── skprompt.txt
        └── config.json
```

### skprompt.txt

```
请为去{{$destination}}旅行的游客推荐{{$interest}}相关的景点。
考虑游客的兴趣是{{$interest}}。
```

### config.json

```json
{
    "schema": 1.1,
    "description": "根据目的地和兴趣推荐旅行景点",
    "input_variables": [
        {
            "name": "destination",
            "description": "旅行目的地",
            "required": true
        },
        {
            "name": "interest",
            "description": "游客的兴趣偏好",
            "required": true
        }
    ]
}
```

### 代码中使用

```csharp
// 加载插件
var pluginsDirectory = Path.Combine(Directory.GetCurrentDirectory(), "Plugins");
var travelPlugin = kernel.ImportPluginFromPromptDirectory(
    Path.Combine(pluginsDirectory, "TravelPlugin")
);

// 调用函数
var result = await kernel.InvokeAsync(
    travelPlugin["Suggestion"],
    new KernelArguments {
        ["destination"] = "日本",
        ["interest"] = "美食和摄影"
    }
);
```

### 行内定义

```csharp
var promptTemplate = @"请为{{$style}}风格的博客写一个开场白：
主题：{{$topic}}。";

var writeFunction = kernel.CreateFunctionFromPrompt(
    promptTemplate: promptTemplate,
    functionName: "WriteBlogIntro",
    description: "写博客开场白"
);

var result = await kernel.InvokeAsync(
    writeFunction,
    new KernelArguments {
        ["topic"] = "人工智能的未来",
        ["style"] = "专业且富有洞察力"
    }
);
```

## 原生函数

### 定义函数

```csharp
public class MathPlugin
{
    [KernelFunction("add")]
    [Description("将两个数字相加并返回结果。")]
    public double Add(
        [Description("要加的第一个数字。")] double number1,
        [Description("要加的第二个数字。")] double number2)
    {
        return number1 + number2;
    }

    [KernelFunction("multiply")]
    [Description("将两个数字相乘并返回结果。")]
    public double Multiply(
        [Description("要乘的第一个数字。")] double number1,
        [Description("要乘的第二个数字。")] double number2)
    {
        return number1 * number2;
    }
}
```

### 注册插件

```csharp
// 方式一：构建时注册
var mathPlugin = new MathPlugin();
kernel.Plugins.AddFromType(mathPlugin, "Math");

// 方式二：依赖注入
builder.Services.AddSingleton<MathPlugin>();
kernel.Plugins.AddFromObject(kernel.Services.GetRequiredService<MathPlugin>(), "Math");
```

### 调用原生函数

```csharp
var result = await kernel.InvokeAsync(
    "Math", "Add",
    new KernelArguments { ["number1"] = 10, ["number2"] = 20 }
);

Console.WriteLine(result.GetValue<double>());  // 输出: 30
```

## 函数链

按顺序执行多个函数：

```csharp
// 步骤1：情感分析
var analyzeResult = await kernel.InvokeAsync(
    "SentimentAnalyzer", "Analyze",
    new KernelArguments { ["input"] = userInput }
);
var sentiment = analyzeResult.GetValue<string>();

// 步骤2：基于情感的回复生成
var responseResult = await kernel.InvokeAsync(
    "ResponseGenerator", "Generate",
    new KernelArguments { ["sentiment"] = sentiment }
);
```

## 自动函数调用

### 配置自动调用

```csharp
var settings = new OpenAIPromptExecutionSettings
{
    ToolCallBehavior = ToolCallBehavior.AutoInvokeKernelFunctions
};

var chatService = kernel.GetRequiredService<IChatCompletionService>();

var result = await chatService.GetChatMessageContentAsync(
    chatHistory: history,
    executionSettings: settings,
    kernel: kernel
);
```

### 完整示例

```csharp
// 1. 注册工具
kernel.Plugins.AddFromType<WeatherPlugin>("Weather");

// 2. 配置自动调用
var settings = new OpenAIPromptExecutionSettings
{
    ToolCallBehavior = ToolCallBehavior.AutoInvokeKernelFunctions
};

// 3. 对话
var history = new ChatHistory();
history.AddUserMessage("北京的天气怎么样？");

var response = await chatService.GetChatMessageContentAsync(
    history,
    executionSettings: settings,
    kernel: kernel
);

Console.WriteLine(response.Content);
```

## 插件设计最佳实践

### 命名规范

| 原则 | 说明 | 示例 |
|------|------|------|
| 清晰命名 | 使用完整无歧义的单词 | `get_user_profile` |
| 动词开头 | 函数表示动作 | `calculate`, `fetch` |
| 描述性 | 名词表示数据 | `user_profile` |

### 函数设计原则

| 原则 | 说明 |
|------|------|
| 单一职责 | 每个函数只做好一件事 |
| 最小参数 | 减少参数数量 |
| 类型明确 | 使用明确的参数类型 |

## 小结

Semantic Kernel 的插件系统提供了强大的扩展能力。语义函数通过提示词模板定义，原生函数通过 C# 代码实现。掌握插件的设计和使用，是构建复杂 AI 应用的基础。
