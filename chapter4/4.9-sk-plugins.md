# 4.9 Semantic Kernel 插件与函数

## 插件与函数的关系

- **插件**：函数的逻辑分组
- **函数**：插件的基本单位

## 语义函数

语义函数由语言模型驱动，通过自然语言描述完成特定任务。

### 定义方式

#### 行内定义

```csharp
var promptTemplate = "请根据下面的主题，创作一个{{$style}}风格的笑话。主题：{{$input}}";

var jokeFunction = KernelFunctionFactory.CreateFromPrompt(
    promptTemplate,
    functionName: "Joke",
    description: "根据一个主题和风格生成一个笑话"
);

var result = await kernel.InvokeAsync(jokeFunction, new KernelArguments
{
    { "input", "程序员" },
    { "style", "冷" }
});
```

#### 标准化构建

使用文件和目录结构定义语义函数：

```
Plugins
└── TravelPlugin
    └── Suggestion
        ├── config.json
        └── skprompt.txt
```

### 加载插件

```csharp
var pluginsDirectory = Path.Combine(Directory.GetCurrentDirectory(), "Plugins");
var travelPlugin = kernel.ImportPluginFromPromptDirectory(
    Path.Combine(pluginsDirectory, "TravelPlugin")
);
```

## 原生函数

原生函数由 C# 代码实现，适合精确计算和资源访问。

### 定义与注册

```csharp
public class MathPlugin
{
    [KernelFunction("add")]
    [Description("将两个数字相加并返回结果。")]
    public double Add(
        [Description("要加的第一个数字。")] double number1,
        [Description("要加的第二个数字。")] double number2)
    {
        return number1 + number2;
    }
}

// 注册函数
builder.Plugins.AddFromType<MathPlugin>("math");
```

## 手动函数调用

通过 `kernel.InvokeAsync()` 实现，需指定插件名、函数名和参数：

```csharp
var result = await kernel.InvokeAsync("lights", "get_light_state", arguments);
```

## 函数链

将多个函数按顺序串联，前一个函数的输出作为后一个函数的输入：

```csharp
// 第 1 步：调用情感分析函数
var sentimentResult = await kernel.InvokeAsync(
    kernel.Plugins["CustomerSupportPlugin"]["AnalyzeSentiment"],
    new KernelArguments { { "input", userInput } }
);

// 第 2 步：将结果作为输入，调用回复生成函数
var finalResult = await kernel.InvokeAsync(
    kernel.Plugins["NativeCustomerSupport"]["GenerateResponseForSentiment"],
    new KernelArguments { { "sentiment", sentimentResult } }
);
```

## 自动函数调用

### 工作原理

1. 将所有可用插件及其描述注册到内核
2. 用户发出自然语言指令后，内核将指令和函数描述发送给语言模型
3. 语言模型分析用户意图决定调用哪些函数
4. Semantic Kernel 解析返回并执行函数调用
5. 执行结果返回给语言模型生成自然语言答复

### 代码示例

```csharp
var settings = new OpenAIPromptExecutionSettings
{
    ToolCallBehavior = ToolCallBehavior.AutoInvokeKernelFunctions
};

var result = await chatCompletionService.GetChatMessageContentAsync(
    history,
    executionSettings: settings,
    kernel: kernel
);
```

## 插件设计最佳实践

1. **清晰命名**：函数和参数的命名应使用完整、无歧义的单词
2. **精确描述**：描述是写给 LLM 的"API 文档"
3. **原子性原则**：尽量让每个函数只做好一件事
4. **参数简化**：减少参数数量，优先使用基元类型
