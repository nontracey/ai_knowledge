# 4.6 LangChain Agent 与 LangGraph

## Agent Executor

### ReAct 框架

ReAct = Reasoning + Acting，通过思考-行动-观察循环解决问题：

```
问题 → 思考 → 行动 → 观察 → (重复直到解决)
```

### Agent Executor 工作流程

```python
from langchain.agents import AgentExecutor, create_react_agent
from langchain.tools import Tool

tools = [search_tool, calculator_tool]

agent = create_react_agent(model, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools)

result = executor.invoke({"input": "用户的复杂问题"})
```

### Agent Executor 的局限

- 执行循环隐式固化
- 缺乏灵活性
- 难以实现复杂行为

## LangGraph

### 为什么需要 LangGraph

LangGraph 通过状态图显式控制流程：

- **控制流明确**：每一步清晰可见
- **状态管理**：灵活的状态共享
- **可定制性**：完全掌控执行逻辑

### 核心概念

| 概念 | 说明 |
|------|------|
| State | 图中共享的数据结构 |
| Node | 工作单元（函数） |
| Edge | 节点间的连接 |
| Conditional Edge | 条件边，根据状态选择流向 |

## 构建 ReAct Agent

### 定义状态

```python
from typing import TypedDict, List, Annotated
import operator

class AgentState(TypedDict):
    messages: Annotated[List, operator.add]
    next_action: str
```

### 定义工具

```python
def search_tool(query: str) -> str:
    """网络搜索工具"""
    return f"搜索结果：{query}的相关信息..."

def calculator(expression: str) -> str:
    """计算器工具"""
    result = eval(expression)
    return str(result)

tools = [search_tool, calculator]
```

### 创建节点

```python
def should_continue(state: AgentState) -> str:
    """判断是否继续"""
    if len(state["messages"]) > 5:
        return "end"
    return "continue"

def call_model(state: AgentState):
    """调用模型"""
    response = model.invoke(state["messages"])
    return {"messages": [response]}

# 构建图
graph = StateGraph(AgentState)
graph.add_node("model", call_model)
graph.add_node("action", execute_tools)

graph.set_entry_point("model")
graph.add_conditional_edges(
    "model",
    should_continue,
    {"continue": "action", "end": END}
)
graph.add_edge("action", "model")

app = graph.compile()
```

## 预构建函数

### create_react_agent

快速创建 ReAct Agent：

```python
from langgraph.prebuilt import create_react_agent

app = create_react_agent(model, tools)

result = app.invoke({"messages": [("user", "复杂问题")]})
```

## 记忆机制

### 检查点保存状态

```python
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
app = graph.compile(checkpointer=checkpointer)

# 第一次对话
result1 = app.invoke(
    {"messages": [("user", "我是小明")]},
    config={"configurable": {"thread_id": "user_1"}}
)

# 第二次对话（带记忆）
result2 = app.invoke(
    {"messages": [("user", "记住我叫什么吗？")]},
    config={"configurable": {"thread_id": "user_1"}}
)
```

## 完整示例

### 对话式 RAG Agent

```python
from langgraph.graph import StateGraph, END

class RAGState(TypedDict):
    question: str
    documents: List
    answer: str
    messages: List

def retrieve(state: RAGState):
    docs = retriever.invoke(state["question"])
    return {"documents": docs}

def generate(state: RAGState):
    context = "\n".join([doc.page_content for doc in state["documents"]])
    response = model.invoke(f"基于以下上下文回答：{context}\n\n问题：{state['question']}")
    return {"answer": response, "messages": [response]}

workflow = StateGraph(RAGState)
workflow.add_node("retrieve", retrieve)
workflow.add_node("generate", generate)
workflow.set_entry_point("retrieve")
workflow.add_edge("retrieve", "generate")
workflow.add_edge("generate", END)

app = workflow.compile()
```

## 小结

LangGraph 提供了比传统 Agent Executor 更强大的工作流控制能力。通过状态图建模，开发者可以精确控制 Agent 的行为，实现复杂的推理和执行流程。
