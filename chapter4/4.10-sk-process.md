# 4.10 Semantic Kernel 流程框架

## 流程框架概述

### 核心思想

从"让 AI 决定做什么"转变为"让 AI 高效执行预定义流程"。

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| Agent 模式 | AI 自主决策 | 探索性任务 |
| 流程模式 | 按固定流程执行 | 企业级业务流程 |

### 与 Agent 框架的对比

| 维度 | 流程框架 | Agent 框架 |
|------|---------|-----------|
| 控制方式 | 确定性 | 动态性 |
| 执行路径 | 预定义 | 运行时决定 |
| 适用场景 | 业务流程 | 复杂推理 |

## 核心概念

### 流程（Process）

端到端的业务目标：

```csharp
var processBuilder = ProcessBuilder.Create("OrderProcessing");
```

### 步骤（Step）

流程中的原子性活动：

```csharp
public class ValidateOrderStep
{
    [KernelFunction]
    [Description("验证订单有效性")]
    public async Task<OrderValidation> ValidateAsync(
        ProcessContext context,
        Order order)
    {
        // 验证逻辑
        var isValid = ValidateOrder(order);
        return new OrderValidation { IsValid = isValid };
    }
}
```

### 事件（Event）

步骤间传递信息的载体：

```csharp
public class OrderValidatedEvent
{
    public string OrderId { get; set; }
    public bool IsValid { get; set; }
}
```

### 事件路由

定义步骤间的流转规则：

```csharp
processBuilder.OnEvent<OrderValidatedEvent>()
    .When(e => e.IsValid)
    .SendTo(processBuilder.GetStep("ProcessOrderStep"));
```

## 构建线性流程

### 定义步骤

```csharp
public class GatherInfoStep
{
    [KernelFunction]
    public string GatherInfo()
    {
        return "产品信息已收集";
    }
}

public class GenerateDocStep
{
    [KernelFunction]
    public string GenerateDoc(string info)
    {
        return $"生成的文档基于: {info}";
    }
}
```

### 编排流程

```csharp
var processBuilder = ProcessBuilder.Create("DocumentWorkflow");

var gatherStep = processBuilder.AddStepFromType<GatherInfoStep>();
var genStep = processBuilder.AddStepFromType<GenerateDocStep>();

// 设置流程
processBuilder
    .StartWith(gatherStep)
    .Then(genStep)
    .WithId("GenerateDocument");

var process = processBuilder.Build();
```

### 执行流程

```csharp
var kernel = Kernel.CreateBuilder()
    .AddOpenAIChatCompletion("gpt-4", apiKey)
    .Build();

var runningProcess = await process.StartAsync(
    new KernelProcessRuntimeContext(kernel)
);

await runningProcess.WaitForCompletionAsync();
```

## 集成 AI 服务

### AI 驱动的步骤

```csharp
public class AIGenerateStep
{
    private readonly Kernel _kernel;

    public AIGenerateStep(Kernel kernel)
    {
        _kernel = kernel;
    }

    [KernelFunction]
    public async Task<string> GenerateAsync(
        ProcessContext context,
        string topic)
    {
        var result = await _kernel.InvokePromptAsync(
            $"为{topic}写一段介绍。"
        );
        return result.ToString();
    }
}
```

### 在流程中使用

```csharp
var aiStep = processBuilder.AddStepFromType<AIGenerateStep>(kernel);
```

## 条件分支

### 基于事件的分支

```csharp
processBuilder.OnEvent<OrderEvent>()
    .When(e => e.Type == OrderType.Standard)
    .SendTo(standardProcessing);

processBuilder.OnEvent<OrderEvent>()
    .When(e => e.Type == OrderType.Priority)
    .SendTo(priorityProcessing);
```

### 审核流程示例

```csharp
// 发布事件
await context.EmitEventAsync(new ProcessEvent("ApprovalRequired")
{
    Data = new { DocumentId = "123" }
});

// 监听事件
processBuilder.OnEvent("ApprovalResult")
    .When(e => e.Data.IsApproved == true)
    .SendTo(publishStep);
```

## 人机协作

### 暂停等待输入

```csharp
public class ApprovalStep
{
    [KernelFunction]
    public async Task<string> RequestApprovalAsync(
        [FromSource] KernelProcessStepContext context,
        Document document)
    {
        // 发送通知给人类审批
        NotifyHuman(document);

        // 暂停，等待人类输入
        var approval = await context.WaitForExternalEventAsync<ApprovalResult>("HumanApproval");

        return approval.IsApproved ? "Approved" : "Rejected";
    }
}
```

## 与 Agent 框架的结合

### 混合模式

Agent 负责规划，流程负责执行：

```
用户请求 → Agent 规划 → 流程执行标准化步骤 → 结果反馈
```

## 小结

Semantic Kernel 流程框架提供了构建确定性业务流程的能力。通过步骤、事件和事件路由，可以设计清晰、可控的 AI 工作流。结合 Agent 框架，可以实现灵活性与确定性的平衡。
