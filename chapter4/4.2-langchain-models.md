# 4.2 LangChain 模型详解

LangChain 的核心是与各种大语言模型进行交互。在这一节中，我们将详细介绍 LangChain 支持的三种模型类型，以及如何在实际项目中使用它们。

## 三种模型类型

### LLM 类——经典文本模型

LLM（Large Language Model）类是最基础的模型接口，采用"文本输入、文本输出"的工作方式。它接收一个字符串作为输入，返回一个字符串作为输出。这种模型是无状态的——每次调用都是独立的，不会记住之前的对话。

**适用场景**：文本补全、翻译、摘要、单次问答等不需要多轮对话的任务。

```python
from langchain_openai import OpenAI

# 创建 LLM 实例
llm = OpenAI(
    model="gpt-3.5-turbo-instruct",
    temperature=0.7
)

# 文本生成
result = llm.invoke("请写一首关于春天的短诗")
print(result)

# 批量生成
results = llm.batch(["什么是机器学习？", "什么是深度学习？"])
for r in results:
    print(r)
    print("---")
```

### Chat 类——对话模型

Chat 模型是现代 AI 应用的核心。它以消息列表的形式进行交互，每条消息都有一个明确的角色标识：

- **SystemMessage**：系统消息，设定 AI 的行为规则和角色定位
- **HumanMessage**：用户消息，代表用户的输入
- **AIMessage**：AI 消息，代表模型的回复

这种设计使得模型能够自然地处理多轮对话和角色扮演。

```python
from langchain_openai import ChatOpenAI
from langchain_core.messages import SystemMessage, HumanMessage, AIMessage

# 创建 Chat 模型实例
chat = ChatOpenAI(
    model="gpt-3.5-turbo",
    temperature=0.7
)

# 单轮对话
messages = [
    SystemMessage(content="你是一位资深的 Python 编程导师，擅长用简单的语言解释复杂的概念。"),
    HumanMessage(content="什么是列表推导式？请举例说明。")
]
response = chat.invoke(messages)
print(response.content)

# 多轮对话——在消息列表中追加历史记录
messages.append(AIMessage(content=response.content))
messages.append(HumanMessage(content="那字典推导式呢？"))

response2 = chat.invoke(messages)
print(response2.content)
```

对话模型的优势在于：
- **上下文感知**：通过消息历史，模型能理解对话的来龙去脉
- **角色设定**：通过系统消息，可以精确控制模型的行为和风格
- **多轮推理**：适合需要反复探讨和深入分析的复杂任务

### Embedding 类——嵌入模型

Embedding 模型的作用是将文本转换成一组数字（向量），这个过程叫做"文本嵌入"。这些向量捕捉了文本的语义信息——意思相近的文本，其向量也相近。

想象一下，Embedding 把每段文本映射到一个高维空间中的某个点。两段文本在语义上越相似，它们对应的点就越接近。这使得"语义搜索"成为可能——不再需要精确匹配关键词，而是通过比较向量距离来找到含义最相近的内容。

```python
from langchain_openai import OpenAIEmbeddings

# 创建 Embedding 模型实例
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 嵌入单个查询文本
query_vector = embeddings.embed_query("什么是机器学习？")
print(f"向量维度: {len(query_vector)}")
print(f"前5个分量: {query_vector[:5]}")

# 嵌入多个文档
doc_vectors = embeddings.embed_documents([
    "机器学习是人工智能的一个分支",
    "深度学习使用神经网络进行学习",
    "今天天气真好，适合出门散步"
])

# 计算相似度（余弦相似度）
import numpy as np

def cosine_similarity(v1, v2):
    return np.dot(v1, v2) / (np.linalg.norm(v1) * np.linalg.norm(v2))

# "机器学习"与"深度学习"的相似度
sim1 = cosine_similarity(doc_vectors[0], doc_vectors[1])
# "机器学习"与"天气"的相似度
sim2 = cosine_similarity(doc_vectors[0], doc_vectors[2])

print(f"机器学习 vs 深度学习: {sim1:.4f}")
print(f"机器学习 vs 天气: {sim2:.4f}")
```

你会发现，"机器学习"与"深度学习"的相似度远高于"机器学习"与"天气"的相似度，尽管它们没有完全相同的词汇。这就是语义搜索的威力。

## 使用 OpenAI 兼容 API

很多模型服务商（如国内的各种大模型平台）提供了兼容 OpenAI 的 API 接口，这意味着你可以用 LangChain 的 OpenAI 集成来访问它们，只需要修改 API 地址和密钥：

```python
from langchain_openai import ChatOpenAI

# 使用兼容 OpenAI 的国产模型
llm = ChatOpenAI(
    model="qwen-plus",                    # 模型名称
    openai_api_key="your-api-key",         # 你的 API 密钥
    openai_api_base="https://dashscope.aliyuncs.com/compatible-mode/v1",  # API 地址
    temperature=0.7
)

response = llm.invoke("你好，请介绍一下你自己")
print(response.content)
```

这种方式让你可以轻松切换不同的模型提供商，而不需要修改大量代码。

## 关键参数详解

理解模型参数对于获得理想的输出至关重要。以下是最常用的参数：

### temperature（温度）

控制输出的随机性，取值范围 0 到 2：

- **0**：最确定性，总是选择概率最高的词。适合事实性问答、代码生成等需要精确答案的场景
- **0.7**：适中的随机性，在保持连贯性的同时有一定的创造性。适合日常对话
- **1.5-2.0**：高随机性，输出更加多样化但也更不可预测。适合创意写作、头脑风暴

```python
# 精确模式——用于事实性问答
precise_llm = ChatOpenAI(temperature=0)

# 创意模式——用于创意写作
creative_llm = ChatOpenAI(temperature=1.2)
```

### top_p（核采样）

另一种控制随机性的方式。模型只会从累积概率达到 top_p 的候选词中采样：

- **0.1**：只考虑概率最高的少数词，输出更保守
- **0.9**：考虑大多数候选词，输出更多样

> **提示**：通常只需要调节 temperature 或 top_p 中的一个，不需要同时调节。

### top_k

限制每次选择时只考虑概率最高的 k 个候选词。top_k = 1 等价于贪心搜索（总是选择概率最高的词），top_k 越大，可选择的范围越广。

### frequency_penalty（频率惩罚）和 presence_penalty（存在惩罚）

这两个参数用于控制模型是否倾向于重复使用相同的词：

- **frequency_penalty**（-2.0 到 2.0）：对已经频繁出现的词施加更大的惩罚，值越高越不容易重复
- **presence_penalty**（-2.0 到 2.0）：只要某个词出现过就施加惩罚，鼓励模型使用新的话题和概念

```python
llm = ChatOpenAI(
    model="gpt-3.5-turbo",
    temperature=0.8,
    frequency_penalty=0.5,   # 减少重复用词
    presence_penalty=0.3     # 鼓励引入新话题
)
```

### max_tokens（最大令牌数）

控制模型输出的最大长度。一个"令牌"大约对应 0.75 个英文单词或半个中文字符。

```python
# 限制简短回复
llm = ChatOpenAI(max_tokens=100)   # 大约 50-75 个中文字

# 允许较长回复
llm = ChatOpenAI(max_tokens=2000)  # 大约 1000-1500 个中文字
```

## 实战示例：构建一个简单的问答助手

让我们把前面学到的知识综合起来，构建一个简单的问答助手：

```python
from langchain_openai import ChatOpenAI
from langchain_core.messages import SystemMessage, HumanMessage, AIMessage

class QAAssistant:
    def __init__(self, system_prompt, model_name="gpt-3.5-turbo", temperature=0.7):
        self.llm = ChatOpenAI(
            model=model_name,
            temperature=temperature
        )
        self.history = [SystemMessage(content=system_prompt)]

    def ask(self, question):
        """提问并获得回答"""
        self.history.append(HumanMessage(content=question))
        response = self.llm.invoke(self.history)
        self.history.append(AIMessage(content=response.content))
        return response.content

    def clear_history(self):
        """清除对话历史"""
        system_prompt = self.history[0].content
        self.history = [SystemMessage(content=system_prompt)]

# 使用示例
assistant = QAAssistant(
    system_prompt="你是一位友善的 AI 助手，用简洁清晰的语言回答问题。如果不确定，请诚实说明。",
    temperature=0.5
)

print(assistant.ask("什么是 LangChain？"))
print("---")
print(assistant.ask("它有哪些主要组件？"))  # 会记住上一轮的上下文
```

这个简单的示例展示了如何使用 Chat 模型的消息机制来维护对话上下文，实现多轮对话。在后续的章节中，我们将在这个基础上不断扩展，加入提示模板、工具调用、链式处理等更高级的功能。
