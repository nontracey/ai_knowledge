# 4.3 工具调用基础

## 什么是工具调用

工具调用让大语言模型能够与外部世界交互，执行实际操作。

### 工具调用的工作原理

```
1. 提供工具蓝图 → 工具列表、名称、功能、参数
2. 大模型推理规划 → 判断是否需要调用工具
3. 生成调用指令 → 结构化 JSON（工具名+参数）
4. 代码执行工具 → 解析 JSON，调用实际函数
5. 结果反馈给大模型 → 包装成消息格式
6. 生成最终回复 → 组织语言输出
```

### 类比理解

```
CEO（大模型）隔离在办公室
    ↓
阅读请求（用户输入）
    ↓
撰写详细指令（JSON）
    ↓
请助理（代码）去执行
    ↓
返回结果给 CEO
    ↓
CEO 整理回复
```

## 工具调用的意义

### 打破数据限制

| 能力 | 说明 | 示例 |
|------|------|------|
| 实时信息 | 获取最新天气、新闻 | 查询明天天气 |
| 私有数据 | 访问内部数据库 | 查询订单状态 |
| 执行操作 | 发送邮件、预订 | 帮用户发邮件 |

### 从"聊天者"到"行动派"

- **传统 LLM**：只能回答问题
- **工具调用 LLM**：能够执行操作

## 常见方法

### 1. 格式化输出要求

在提示词中指定输出格式：

```
请按以下 JSON 格式返回：
{
  "function": "get_weather",
  "parameters": {
    "city": "北京"
  }
}
```

### 2. 少样本学习

提供示例：

```
示例1：
输入：查询上海天气
输出：{"function": "get_weather", "parameters": {"city": "上海"}}

示例2：
输入：北京今天多少度
输出：{"function": "get_weather", "parameters": {"city": "北京"}}
```

### 3. 严格指令

```
重要：请只输出 JSON 格式，不要有任何解释性文字。
```

## 实现原理

### 内置函数描述

以结构化数据（如 JSON Schema）描述函数：

```json
{
  "name": "get_weather",
  "description": "获取指定城市的天气信息",
  "parameters": {
    "type": "object",
    "properties": {
      "city": {
        "type": "string",
        "description": "要查询天气的城市名称"
      }
    },
    "required": ["city"]
  }
}
```

### 结构化输出

现代模型支持原生函数调用：

```python
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "北京天气怎么样？"}],
    tools=[{
        "type": "function",
        "function": {
            "name": "get_weather",
            "parameters": {"type": "object", "properties": {...}}
        }
    }]
)
```

## 天气查询示例

### 提示词设计

```python
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的天气信息",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名称，如北京、上海"
                    },
                    "date": {
                        "type": "string",
                        "description": "日期，如今天、明天"
                    }
                },
                "required": ["city"]
            }
        }
    }
]
```

### 处理函数调用

```python
response = client.chat.completions.create(
    model="gpt-4",
    messages=messages,
    tools=tools
)

# 获取工具调用
if response.choices[0].message.tool_calls:
    tool_call = response.choices[0].message.tool_calls[0]
    function_name = tool_call.function.name
    arguments = json.loads(tool_call.function.arguments)

    # 执行工具
    if function_name == "get_weather":
        result = get_weather(arguments["city"], arguments.get("date"))
```

## 工具描述最佳实践

### 命名规范

| 差 | 好 | 更好 |
|---|-----|-----|
| data | get_user_profile | fetch_user_profile_by_email |

### 功能描述

| 差 | 好 | 更好 |
|---|-----|-----|
| 搜索工具 | 执行网络搜索 | 当用户提出关于XXXX问题时，使用此工具进行网络搜索以获取最新答案 |

### 参数描述

```python
# 差
city: str

# 好
city: str = Field(description="用户想要查询天气的城市名称")

# 更好
city: str = Field(description="用户想要查询天气的城市名称，例如 beijing, shanghai, wuhan，应使用拼音")
```

## 流式工具调用

```python
stream = client.chat.completions.create(
    model="gpt-4",
    messages=messages,
    tools=tools,
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.tool_calls:
        # 实时展示工具调用过程
        print(chunk.choices[0].delta.tool_calls[0])
```

## 小结

工具调用是 Agent 能力的基础，通过结构化的函数描述和调用机制，让 LLM 能够执行实际操作。掌握工具调用的原理和最佳实践，是构建智能 Agent 的关键技能。
