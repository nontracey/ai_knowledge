# 4.3 工具调用基础

## 工具调用概述

工具调用是让大模型能够使用外部工具的能力，扩展其行动边界。

### 工作原理

1. 提供工具蓝图：展示工具列表、工具的模式、工具的说明书
2. 大模型进行推理和规划：根据用户输入判断需要调用哪个工具
3. 生成调用指令：生成结构化的 JSON 对象
4. 我们的代码负责执行：解析 JSON 对象，调用实际函数
5. 将结果反馈给大模型
6. 大模型生成最终的回复

### 类比说明

如同 CEO（ 大模型）在办公室隔离 → 阅读请求 → 撰写详细指令 JSON → 请助理（代码）去执行 → 把结果拿回来。

## 常见方法

### 格式化输出要求

在提示词中明确要求模型按特定格式（如 JSON）返回结果。

### 少样本学习示例

通过在提示词中提供少量标准答案示例，让模型学习输出格式。

### 严格的指令和约束

增加详尽的约束条件，防止模型自由发挥。

### 后处理步骤

通过正则表达式提取、JSON 合法性检查等手段过滤整理输出。

## 天气查询示例

设计提示词引导模型返回结构化的 JSON：

```json
{
  "function": "get_weather",
  "parameters": {
    "location": "北京",
    "date": "今天"
  }
}
```

## 工具调用的优势

- **打破数据限制**：可获取实时信息
- **成为懂业务的专属助理**：与私有数据交互
- **从"聊天者"蜕变为"行动派"**：执行真实操作

## 工具描述的最佳实践

### 命名规范

- 使用清晰、面向行动的命名
- 差：`data`
- 好：`get_user_profile`

### 描述规范

- 在描述中明确工具何时使用
- 为参数提供详细的描述和示例

### 单一职责

- 保持工具功能的单一和专注
- 多个小工具比一个大工具更好
