# 4.7 LangChain RAG 应用开发

## RAG 是什么

检索增强生成（Retrieval-Augmented Generation），通过检索外部知识源增强 LLM 知识储备和生成能力。

### RAG 工作原理

- **索引阶段**：加载 → 分割 → 嵌入 → 存储
- **检索与生成阶段**：检索 → 生成

### RAG 解决的问题

- 知识过时、内容幻觉
- 高时效性、高准确性需求

## LangChain 核心价值

LangChain 解决了从零开发 RAG 应用面临的诸多难题：

1. 繁杂的数据接入
2. 复杂的文本处理
3. 多样的模型与接口
4. 专业的向量存储
5. 粘合所有环节

## RAG 完整流程与组件

### 1. 加载（Load）

文档加载器将各种数据源加载为文档对象，包含 `page_content` 和 `metadata`。

常用加载器：
- **TextLoader**：纯文本文件
- **WebBaseLoader**：网页内容
- **CSVLoader**：CSV 文件
- **PyPDFLoader**：PDF 文件
- **DirectoryLoader**：目录下的多个文件

### 2. 分割（Split）

文本分割将长文档切成小片段（chunk），便于嵌入和检索。

核心参数：
- **块大小（chunk_size）**：文本块的目标最大尺寸
- **块重叠（chunk_overlap）**：连续块之间重复内容的长度

分割器类型：
- **CharacterTextSplitter**：按分隔符分割
- **RecursiveCharacterTextSplitter**：递归分割，语义更连贯
- **TokenTextSplitter**：按 Token 分割，精确控制大小
- **MarkdownTextSplitter**：按 Markdown 标题分割
- **CodeTextSplitter**：按代码结构分割

### 3. 嵌入（Embed）

将文本片段转换成向量。

```python
from langchain_huggingface import HuggingFaceEmbeddings

embeddings = HuggingFaceEmbeddings(model_name="sentence-transformers/all-MiniLM-L6-v2")
```

### 4. 存储（Store）

使用向量数据库存储向量和原文，建立索引。

常用向量数据库：
- **FAISS**：高性能相似性搜索库
- **Chroma**：为 AI 应用设计的向量数据库
- **Pinecone**：云原生向量数据库服务
- **Milvus**：开源向量数据库

### 5. 检索（Retrieve）

根据用户问题从向量数据库中找出最相关的文本。

检索类型：
- **相似性搜索**：返回最接近的 k 个文档
- **MMR（最大边际相关性）**：平衡相关性和多样性
- **相似性得分阈值**：设置最低相似度要求

### 6. 生成（Generate）

把检索到的资料和用户问题一起交给大模型生成回答。

## 构建完整 RAG 链

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_openai import ChatOpenAI

# 创建向量存储
vectorstore = FAISS.from_documents(documents=splits, embedding=embeddings)
retriever = vectorstore.as_retriever()

# 创建提示模板
prompt = ChatPromptTemplate.from_messages([
    ("system", "根据以下上下文回答问题。\n\n{context}"),
    ("human", "{question}")
])

# 创建链
llm = ChatOpenAI(model="gpt-4")
rag_chain = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt
    | llm
)

result = rag_chain.invoke("什么是人工智能？")
```

## 对话式 RAG

无状态 RAG 无法处理连续追问，需要引入对话历史和查询重写。

### LangGraph 构建对话式 RAG

使用 LangGraph 实现：
1. **rewrite_node**：查询重写节点
2. **retrieve_node**：检索节点
3. **generate_node**：生成节点
4. **agent_router**：条件路由

## 两阶段高级检索策略

### 召回阶段

使用多查询检索器增加召回率。

### 精排阶段

使用交叉编码器（Cross-Encoder）对召回结果进行重排。
