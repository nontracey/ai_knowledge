# 4.8 Semantic Kernel 概述与环境搭建

## 与 LangChain 对比

| 特性 | LangChain | Semantic Kernel |
|------|-----------|-----------------|
| 设计理念 | 社区驱动，快速迭代 | 企业级应用，注重稳定性 |
| 适用场景 | 快速开发和试验 | 企业级应用 |
| 编程语言 | Python 为主 | C# 和 Python |
| 类型安全 | 动态类型 | 强类型 |

## 架构概览

### 内核（Kernel）

Semantic Kernel 的核心，负责接收请求、编排插件、管理服务和执行任务。

- **服务容器**：语言模型服务、日志服务，依赖注入模式
- **执行引擎**：函数调用 AI 服务

### 插件（Plugins）

功能扩展的核心机制，包含语义函数和原生函数。

### 记忆（Memory）

使 AI 应用能够保持上下文，支持连贯对话和复杂任务处理。

## 环境准备

### 创建项目

创建 .NET 控制台应用程序，添加 Semantic Kernel 的 NuGet 包。

### 配置服务

通过配置连接器，将内核连接到大语言模型服务。

### 密钥安全性

使用 .NET 用户机密（User Secrets）管理 API Key：

```bash
dotnet user-secrets init
dotnet user-secrets set "Qianwen:ApiKey" "your-api-key"
```

### 构建内核

```csharp
var builder = Kernel.CreateBuilder();

builder.Services.AddLogging(c => c.AddConsole().SetMinimumLevel(LogLevel.Information));
builder.AddOpenAIChatCompletion(
    modelId: "qwen-plus",
    apiKey: apiKey,
    endpoint: new Uri("https://dashscope.aliyuncs.com/compatible-mode/v1")
);

var kernel = builder.Build();
```
